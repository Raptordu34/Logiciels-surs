Exercice 3 — Opérations sur les relations

Fondamental ◎ 10 pts ⏱ 20–30 min
Objectifs
  • Calculer le domaine d'une relation (dom)
  • Appliquer la restriction de domaine (◁) et de codomaine (▷)
  • Maîtriser l'opération de surcharge (←)

Prérequis : Cours 2 — Relations, fonctions et notations Z

Contexte — Opérations sur une relation

Soit A = {a1, a2, a3} et B = {a1, a2, a3}. On considère la relation R ⊆ A × B définie par :

R = { a1 → a2, a1 → a3, a3 → a2 }

Calculez le résultat des expressions suivantes :

Rappel de notation
  • x → y désigne le couple (x, y) — aussi appelé maplet en notation Z.
  • dom(R) : ensemble des antécédents de R (éléments de gauche de chaque paire).
  • S ◁ R : restriction de R au domaine S — ne conserve que les paires dont la source ∈ S.
  • R ▷ T : restriction de R au codomaine T — ne conserve que les paires dont la cible ∈ T.
  • R ← S : surcharge de R par S — S écrase R là où S est défini.

Pour R ⊆ A × B, S ⊆ A et T ⊆ B :

  • dom(R) = { x | ∃ y · x → y ∈ R } — éléments qui ont au moins une image dans R.
  • S ◁ R = { x → y ∈ R | x ∈ S } — filtre les paires dont la source est dans S.
  • R ▷ T = { x → y ∈ R | y ∈ T } — filtre les paires dont la cible est dans T.
  • R ← S = (dom(S) ⩤ R) ∪ S — supprime de R les paires de source dans dom(S), puis ajoute toutes les paires de S.

Relations vs fonctions : une relation peut associer plusieurs images à un même antécédent. Ici, a1 → a2 et a1 → a3 coexistent — R n'est donc pas une fonction partielle.

◁ (restriction de domaine) : S ◁ R supprime toutes les paires dont la source n'appartient pas à S. Si S = {a1}, seules les paires commençant par a1 survivent.

▷ (restriction de codomaine) : R ▷ T supprime les paires dont la cible n'appartient pas à T. Si T = {a2}, seules les paires finissant par a2 survivent.

← (surcharge) : R ← S fonctionne comme une mise à jour globale. On retire toutes les paires de R ayant une source dans dom(S), puis on ajoute l'intégralité de S.

Point clé

La surcharge agit sur le domaine entier de S. Si S = { a1 → a1 }, alors toutes les paires de R dont la source est a1 — y compris a1 → a2 et a1 → a3 — sont remplacées d'un coup.

Exemple

Soit P = { x → y, x → z, w → y }.

{ x } ◁ P = { x → y, x → z } (la paire w → y est supprimée car w ∉ {x})

P ▷ { y } = { x → y, w → y } (la paire x → z est supprimée car z ∉ {y})

Analogie — surcharge

Intuition : imaginez un carnet de contacts. R est l'ancien carnet, S est une liste de mises à jour. Pour chaque contact présent dans S, on efface toutes ses anciennes entrées dans R et on applique la nouvelle valeur. Les contacts absents de S restent intacts.

Q1 2 pts Fondamental

a. Calculez dom({ a1 → a2, a1 → a3, a3 → a2 }).

Piège courant : Inclure les images (éléments de droite) dans le domaine. Le domaine rassemble uniquement les antécédents — les éléments de gauche. Même si a1 apparaît dans deux paires, il ne figure qu'une seule fois dans dom(R).
Parcourez R paire par paire et notez l'élément de gauche : a1 → a2 donne a1 ; a1 → a3 donne a1 (doublon — on ne l'ajoute pas deux fois) ; a3 → a2 donne a3. L'élément a2 n'est jamais à gauche d'une paire.

Solution — Q1

  • Paire a1 → a2 : antécédent = a1
  • Paire a1 → a3 : antécédent = a1 (déjà présent, doublon ignoré)
  • Paire a3 → a2 : antécédent = a3
dom({ a1 → a2, a1 → a3, a3 → a2 }) = { a1, a3 }

L'élément a2 n'appartient pas au domaine : il n'apparaît qu'en position d'image. Le fait que a1 intervienne dans deux paires ne le compte qu'une seule fois dans l'ensemble résultat.

Q2 4 pts Intermédiaire

b. Calculez { a1 } ◁ { a1 → a2, a1 → a3, a3 → a2 } ▷ { a2 }.

Piège courant : N'appliquer qu'une seule des deux restrictions. L'expression combine ◁ (filtre sur la source) et ▷ (filtre sur la cible) — les deux filtres doivent être appliqués pour obtenir le résultat correct.
Commencez par la restriction de domaine : { a1 } ◁ R conserve uniquement les paires dont la source est a1. Parmi { a1 → a2, a1 → a3, a3 → a2 }, quelles paires survivent ?
Après { a1 } ◁ R = { a1 → a2, a1 → a3 }, appliquez ▷ { a2 } : ne conservez que les paires dont la cible est a2. Laquelle des deux paires restantes a bien a2 à droite ?

Solution — Q2

  • Étape 1 — restriction de domaine ◁ : ne conserver que les paires avec a1 à gauche.
    { a1 } ◁ { a1 → a2, a1 → a3, a3 → a2 } = { a1 → a2, a1 → a3 }
    (la paire a3 → a2 est supprimée car a3 ∉ { a1 })
  • Étape 2 — restriction de codomaine ▷ : ne conserver que les paires avec a2 à droite.
    { a1 → a2, a1 → a3 } ▷ { a2 } = { a1 → a2 }
    (la paire a1 → a3 est supprimée car a3 ∉ { a2 })
{ a1 } ◁ R ▷ { a2 } = { a1 → a2 }

Les deux restrictions se combinent : on exige simultanément que la source soit dans {a1} et que la cible soit dans {a2}. Seule la paire a1 → a2 satisfait les deux conditions.

Q3 4 pts Avancé

c. Calculez { a1 → a2, a1 → a3, a3 → a2 } ← { a1 → a1 }.

Le symbole ← désigne l'opération de surcharge.

Piège courant : Ne remplacer qu'une seule paire de R au lieu de toutes celles qui partagent le même antécédent. La surcharge supprime l'ensemble des paires de R dont la source appartient à dom(S). Ici, a1 → a2 et a1 → a3 sont toutes les deux supprimées — pas seulement la première.
La surcharge R ← S se décompose en deux étapes :
  1. Retirer de R toutes les paires dont la source est dans dom(S).
  2. Ajouter toutes les paires de S au résultat.
Commencez par calculer dom({ a1 → a1 }).
dom({ a1 → a1 }) = { a1 }. Supprimez de R toutes les paires de source a1 : disparaissent a1 → a2 et a1 → a3. Il reste { a3 → a2 }. Ajoutez-y { a1 → a1 }.

Solution — Q3

  • dom({ a1 → a1 }) = { a1 } — la mise à jour porte sur l'antécédent a1.
  • Anti-restriction : supprimer de R toutes les paires de source a1.
    Supprimées : a1 → a2 et a1 → a3 (les deux paires ayant a1 à gauche).
    Il reste : { a3 → a2 }.
  • Union : ajouter les paires de S = { a1 → a1 }.
    Résultat = { a3 → a2 } ∪ { a1 → a1 }.
R ← { a1 → a1 } = { a1 → a1, a3 → a2 }

La surcharge a remplacé les deux paires a1 → a2 et a1 → a3 par une seule paire a1 → a1. La paire a3 → a2 reste inchangée car a3 ∉ dom({ a1 → a1 }).

Pour aller plus loin
  • Que vaut ∅ ◁ R ? Et A ◁ R (avec A = domaine complet) ? Ces cas limites révèlent le comportement de la restriction sur les extrêmes.
  • La surcharge est-elle commutative ? Construisez un contre-exemple montrant que R ← S ≠ S ← R en général.
  • Exprimez formellement la surcharge à l'aide de l'anti-restriction de domaine (⩤) et de l'union ensembliste — comparez avec la définition vue en cours.
