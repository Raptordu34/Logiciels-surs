Exercice 2 — Compréhension – Extension

Intermédiaire ◎ 12 pts ⏱ 30–40 min
Objectifs
  • Convertir une description en compréhension en liste en extension (et vice versa)
  • Distinguer une relation d'une fonction à partir de sa définition
  • Écrire une fonction avec la notation λ

Prérequis : Cours 1 — Ensembles · Cours 2 — Relations et fonctions, notation λ

Consigne

Pour chaque ensemble suivant (décrit en compréhension), fournir une description en extension. Si la relation décrite est une fonction, l'exprimer aussi en utilisant la notation λ.

Description en extension : on liste explicitement tous les éléments. Exemple : {0, 2, 4, 6, 8, 10}. Si l'ensemble est infini, on montre quelques éléments avec des points de suspension.

Description en compréhension : on donne une propriété caractéristique. Exemple : {x ∈ ℕ | x < 11 ∧ x mod 2 = 0}. La compréhension est plus concise pour les ensembles infinis ou très grands.

Relation de A vers B : tout sous-ensemble de A×B. Un couple (a, b) appartient à la relation si a est relié à b.

Fonction de A vers B : relation où chaque élément de A a exactement une image dans B. Critère : ∀ a ∈ A, il existe un unique b tel que (a, b) ∈ f.

Notation λ : λx ∈ A . expr(x) désigne la fonction qui associe à tout x de A la valeur expr(x). Exemple : λx ∈ ℕ . x² associe à chaque entier naturel son carré.

Critère de fonction

Une relation R ⊆ A×B est une fonction si et seulement si : pour tout a ∈ A, il existe un et un seul b ∈ B tel que (a, b) ∈ R.

  • Si un a est relié à deux b distincts → pas une fonction (ambiguïté).
  • Si un a n'est relié à aucun b → fonction partielle, non définie sur tout A.

Exemple

Compréhension : {x ∈ ℕ | x < 5}
Extension : {0, 1, 2, 3, 4}

Compréhension : {(n, n²) | n ∈ ℕ}
Extension (partielle) : {(0,0), (1,1), (2,4), (3,9), …}
Notation λ : λn ∈ ℕ . n²

Analogie

Intuition : la compréhension est une recette (« les entiers pairs inférieurs à 11 »), l'extension est le plat servi ({0, 2, 4, 6, 8, 10}). La notation λ est une machine à calculer : λx . x/2 est la machine « prends x, rends x divisé par 2 ».

Q1 3 pts Fondamental

Soit la relation définie en compréhension :

{ (x, y) ∈ ℕ² | x < 5 ∧ x + y = 3 }

Donner une description en extension (complète si l'ensemble est fini). La relation est-elle une fonction ? Si oui, exprimer avec la notation λ.

Piège courant : Oublier que y doit aussi appartenir à ℕ. La contrainte x + y = 3 impose y = 3 − x. Pour que y ≥ 0, il faut x ≤ 3. Ainsi x = 4 est exclu même si x < 5 : le couple (4, −1) n'est pas dans ℕ². L'ensemble est donc fini, avec exactement 4 couples.
La contrainte x + y = 3 détermine y de façon unique dès que x est fixé : y = 3 − x. Testez les valeurs x = 0, 1, 2, 3, 4 (qui vérifient x < 5 et x ∈ ℕ) et vérifiez pour chacune si y = 3 − x est bien dans ℕ.

x=0 → y=3 ✓ ; x=1 → y=2 ✓ ; x=2 → y=1 ✓ ; x=3 → y=0 ✓ ; x=4 → y=−1 ✗ (pas dans ℕ).

L'ensemble complet est {(0,3), (1,2), (2,1), (3,0)}. Chaque valeur de x donne exactement une valeur de y → c'est bien une fonction.

Solution — Q1

Description en extension (complète — ensemble fini) :

{ (0, 3), (1, 2), (2, 1), (3, 0) }

Pour tout x ∈ ℕ avec x < 5, on calcule y = 3 − x. La contrainte y ∈ ℕ (y ≥ 0) impose x ≤ 3 — le cas x = 4 donne y = −1 ∉ ℕ et est donc exclu. L'ensemble est fini (4 couples).

Est-ce une fonction ? Oui

Pour tout x ∈ {0, 1, 2, 3}, la valeur y = 3 − x est déterminée de façon unique par la contrainte x + y = 3. Chaque antécédent a exactement une image : c'est une fonction.

Notation λ :

λx ∈ { 0, 1, 2, 3 } . (3 − x)

Le domaine est fini et peut être listé en extension. On aurait aussi pu l'écrire en compréhension : {z ∈ ℕ | z ≤ 3}.

Q2 3 pts Fondamental

Soit la relation définie en notation λ :

λx ( x ∈ ℕ ∧ x < 11 ∧ x mod 2 = 0 | x / 2 )

Identifier le domaine, donner la liste complète des couples en extension, et confirmer que c'est une fonction.

Piège courant : Oublier 0. En ℕ, 0 est inclus par convention dans ce cours. Or 0 est pair et 0 < 11 — le couple (0, 0) fait partie de la relation. Sans lui, on n'obtient que 5 couples au lieu de 6.
Identifiez d'abord le domaine : les x ∈ ℕ vérifiant x < 11 et x mod 2 = 0. Listez-les, puis calculez x/2 pour chacun afin d'obtenir les couples (x, x/2).

Q3 3 pts Intermédiaire

Soit la fonction décrite en langage naturel :

« Une fonction qui associe n avec 1 + 2n »

Donner une description en compréhension, une description en extension (partielle), et la notation λ.

Piège courant : Confondre 1 + 2n avec (1 + 2)n = 3n. L'exposant porte uniquement sur 2 — l'expression est bien 1 + (2n). Pour n = 3 : 1 + 8 = 9, pas 33 = 27.
Le domaine naturel de cette fonction est ℕ (entiers naturels). Calculez les premières images pour n = 0, 1, 2, 3, 4 afin de construire l'extension partielle.

Calculs : n=0 → 1+1=2 ; n=1 → 1+2=3 ; n=2 → 1+4=5 ; n=3 → 1+8=9 ; n=4 → 1+16=17.

En compréhension : {(n, 1+2n) | n ∈ ℕ}. Notation λ : λn ∈ ℕ . (1 + 2n).

{ (n, 1 + 2ⁿ) | n ∈ ℕ }

Q4 3 pts Avancé

Soit la fonction décrite en langage naturel :

« Une fonction qui associe un ensemble non vide d'entiers naturels avec son plus petit membre »

Identifier le domaine, donner une description en compréhension, quelques exemples en extension, et la notation λ.

Piège courant : Oublier la condition « non vide ». L'ensemble vide ∅ n'a pas de plus petit élément — la fonction serait non définie sur ∅. Le domaine doit donc explicitement exclure ∅, noté P*(ℕ) = P(ℕ) \ {∅}.

Le domaine est l'ensemble de tous les sous-ensembles non vides de ℕ, noté P*(ℕ).

Pour trouver l'image d'un élément du domaine (qui est lui-même un ensemble), il suffit de prendre son minimum — qui existe toujours pour tout sous-ensemble non vide de ℕ (principe du bon ordre).

Quelques couples : ({0}, 0), ({1, 2}, 1), ({3, 5, 7}, 3), ({100}, 100).

En compréhension : {(A, m) | A ∈ P*(ℕ) ∧ m = min(A)}. Notation λ : λA ∈ P*(ℕ) . min(A).

Solution — Q4

Domaine : P*(ℕ) = P(ℕ) \ {∅} — l'ensemble de tous les sous-ensembles non vides de ℕ.

Codomaine : ℕ.

Description en compréhension :

{ (A, m) | A ∈ P*(ℕ)  ∧  m = min(A) }

Exemples en extension :

{ ({0}, 0),  ({1}, 1),  ({1, 2, 3}, 1),  ({3, 5, 7}, 3),  ({100}, 100),  … }

Pourquoi min(A) existe-t-il toujours ? Par le principe du bon ordre : tout sous-ensemble non vide de ℕ admet un plus petit élément. C'est une propriété fondamentale de ℕ — elle ne se transfère pas à ℤ (les entiers relatifs n'ont pas de minimum).

Notation λ :

λA ∈ P*(ℕ) . min(A)

Est-ce une fonction ? Oui

Pour tout A ∈ P*(ℕ), min(A) est unique — un ensemble ne peut avoir qu'un seul plus petit élément. La condition de fonction est satisfaite. Notez cependant que deux antécédents distincts peuvent partager la même image : ({1}, 1) et ({1, 2, 3}, 1) → la fonction n'est pas injective.
