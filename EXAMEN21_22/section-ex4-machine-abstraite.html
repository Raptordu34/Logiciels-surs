Exercice 4 — Machine Abstraite

Avancé ◎ 4 pts ⏱ 20–30 min
Objectifs
  • Analyser la préservation d'un invariant dans une machine B complète
  • Simuler l'exécution séquentielle d'opérations non déterministes

Prérequis : Cours — Machines abstraites B, substitution ANY, SELECT

Contexte — Machine q2

Soit la machine abstraite suivante :

MACHINE q2
VARIABLES v1
INVARIANT v1 ∈ 0..4
INITIALISATION v1 := 0
OPERATIONS

res ← op1 =
  PRE v1 ≤ 2 THEN
    ANY x1 WHERE x1 ∈ NAT ∧ x1 ∈ 0..2
    THEN res := x1 || v1 := v1 + x1 END
  END;

res ← op2 =
  PRE v1 ∈ NAT THEN
    ANY x1 WHERE x1 ∈ NAT ∧ x1 ∈ 0..2
    THEN res := x1 || v1 := v1 + x1 END
  END;

op3 =
  PRE v1 ∈ 0..2 THEN
    SELECT v1 = 0 THEN v1 := v1 + 1
    WHEN v1 = 1 THEN v1 := 0
    WHEN v1 = 2 THEN v1 := 1
    ELSE v1 := 3 END
  END

END
Note : NAT désigne l'ensemble des naturels (ℕ). L'invariant impose v1 ∈ 0..4, c'est-à-dire v1 ∈ {0, 1, 2, 3, 4}.

ANY x WHERE P THEN S END : choisit non-déterministiquement un x satisfaisant P, puis exécute S. Pour prouver la préservation de l'invariant, il faut que S préserve l'invariant pour tout x satisfaisant P.

SELECT c₁ THEN S₁ WHEN c₂ THEN S₂ … ELSE S END : exécute la branche dont la garde est vraie. La clause ELSE ne s'exécute que si aucune garde n'est vraie.

PRE P THEN S END : la précondition P est une obligation pour l'appelant. Si l'appelant ne la respecte pas, le comportement est indéfini. Cependant, pour la préservation de l'invariant, on doit prouver : I ∧ PRE ⇒ [S]I.

Q1 2 pts Intermédiaire

Indiquer si l'invariant v1 ∈ 0..4 est préservé par chaque opération. Justifier votre réponse.

Piège courant : Ne pas confondre la précondition (obligation de l'appelant) avec la vérification de l'invariant. Même si la PRE est faible, il faut vérifier que l'invariant est préservé sous I ∧ PRE.
Pour chaque opération, identifiez les valeurs maximales de v1 autorisées par I ∧ PRE, puis les valeurs maximales de x1 autorisées par le ANY. Calculez la valeur maximale de v1 après exécution.
Attention à op2 : sa précondition est v1 ∈ NAT, qui n'impose aucune borne supérieure. Sous l'invariant, v1 peut valoir jusqu'à 4. Que se passe-t-il si v1 = 4 et x1 = 2 ?

Solution — Q1

L'invariant n'est PAS préservé, à cause de l'opération op2.

op1 : préserve l'invariant ✓

  • PRE : v1 ≤ 2, donc v1 ∈ {0, 1, 2}
  • ANY : x1 ∈ 0..2, donc x1 ∈ {0, 1, 2}
  • Valeur max de v1' = 2 + 2 = 4 ✓
  • Valeur min de v1' = 0 + 0 = 0 ✓

op2 : viole l'invariant ✗

  • PRE : v1 ∈ NAT (pas de borne supérieure)
  • Sous I ∧ PRE : v1 ∈ 0..4
  • ANY : x1 ∈ 0..2
  • Si v1 = 4 et x1 = 1 → v1' = 5 ∉ 0..4 ✗
  • Si v1 = 4 et x1 = 2 → v1' = 6 ∉ 0..4 ✗

op3 : préserve l'invariant ✓

  • PRE : v1 ∈ 0..2
  • v1 = 0 → v1' = 1 ✓
  • v1 = 1 → v1' = 0 ✓
  • v1 = 2 → v1' = 1 ✓
  • ELSE : impossible car les 3 gardes couvrent 0..2
Q2 2 pts Intermédiaire

Quel est l'état de la machine après avoir exécuté les 3 opérations suivantes :

1 ← op1 ; 2 ← op1 ; 2 ← op2

Par exemple, 1 ← op1 signifie que l'opération a retourné la valeur 1 en sortie après exécution.

La valeur retournée (res) correspond à la valeur de x1 choisie par le ANY. Si 1 ← op1, alors x1 = 1.
Partez de v1 = 0 (initialisation) et appliquez séquentiellement : v1 := v1 + x1 pour chaque opération avec le x1 déduit de la valeur retournée.

Solution — Q2

État initial : v1 = 0

ÉtapeOpérationres = x1v1 avantv1 après (v1 + x1)
11 ← op1100 + 1 = 1
22 ← op1211 + 2 = 3
32 ← op2233 + 2 = 5

État final : v1 = 5

Remarque : La valeur v1 = 5 viole l'invariant v1 ∈ 0..4, ce qui confirme que op2 ne préserve pas l'invariant (comme montré en Q1).
