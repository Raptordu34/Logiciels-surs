Exercice 4 — Invariant et Variant

Intermédiaire ◎ 6 pts ⏱ 20 min
Objectifs
  • Démontrer la correction d'un programme avec boucle.
  • Identifier un invariant de boucle à partir d'une postcondition.
  • Proposer un variant garantissant la terminaison du programme.

Prérequis : Triplets de Hoare, sémantique de la boucle while, correction totale

Contexte — Multiplication russe

Le programme suivant calcule le produit de deux entiers naturels a et b uniquement par additions et divisions entières, sans jamais utiliser la multiplication directement. C'est l'algorithme dit de multiplication russe (ou multiplication binaire). 

-- Précondition : a ∈ ℕ, b ∈ ℕ
x     := a;
y     := b;
total := 0;

while x > 0 do
    if x mod 2 = 1 then
        total := total + y;
        x     := x - 1
    end;
    x := x / 2;   -- division entière (ex : 7/2 = 3)
    y := y * 2
end

-- Postcondition : total = a × b
Notations :
  • x mod 2 vaut 0 si x est pair, 1 si x est impair.
  • x / 2 désigne la division entière (quotient sans reste).

Données du problème — l'énoncé fournit :

  • Invariant : I ≡ x ∈ ℕ ∧ total + x × y = a × b
  • Variant : V = x
  • Postcondition : total = a × b

Le but de l'exercice est de prouver la correction du programme en vérifiant les trois propriétés de l'invariant (établissement, conservation, conclusion) et la terminaison par le variant.

Pourquoi prouver la correction d'un programme ?

Tester un programme, c'est vérifier qu'il marche sur quelques exemples. Mais ça ne prouve rien de général : un bug peut se cacher dans un cas que tu n'as pas testé.

Prouver la correction, c'est démontrer mathématiquement que le programme donne toujours le bon résultat, quels que soient les inputs. C'est ce qu'on fait en logiciel sûr : on ne se contente pas de tester, on prouve.

Exemple : l'algorithme de multiplication russe de cet exercice ne fait jamais de multiplication directe. Comment être sûr qu'il calcule bien a × b dans tous les cas ? Seule une preuve formelle le garantit.

Le problème spécifique des boucles while

Pour une séquence d'instructions sans boucle, prouver la correction est relativement simple : on remonte les substitutions avec le wp, instruction par instruction.

Mais une boucle while pose un problème fondamental : on ne sait pas à l'avance combien de fois elle va s'exécuter. Elle peut tourner 3 fois, 100 fois, 0 fois… On ne peut pas dérouler le calcul à l'infini.

La solution : au lieu de raisonner sur toutes les itérations, on cherche une propriété qui reste vraie à chaque instant, quel que soit le tour. C'est l'invariant.

La "grosse artillerie" est réservée aux boucles while — et uniquement elles

L'invariant I, le variant V, et les 4 obligations de preuve avec δ sont des outils exclusivement conçus pour les boucles while. Pourquoi ? Parce que c'est la seule structure de programmation dont on ne sait pas à l'avance combien de fois le code va s'exécuter. La boucle peut tourner 0 fois, 3 fois, 100 fois… On ne peut donc pas dérouler mécaniquement le calcul jusqu'à la fin.

Pour toutes les autres structures — séquences et conditions if — on sait exactement ce qui va s'exécuter. On peut remonter la précondition wp instruction par instruction, sans aucun invariant ni aucun variant.

Structure Nb d'exécutions connu ? Méthode de preuve Invariant ? Variant ?
S₁ ; S₂ ; … ; Sₙ (séquence) Oui — 1 fois chaque instruction wp de droite à gauche, substitution mécanique Non Non
if B then S₁ else S₂ Oui — exactement une des deux branches wp des deux branches séparément + conjonction Non Non
while E do S end Non — 0, 1, ou ∞ tours possibles Invariant I + Variant V + 4 obligations (δ) Oui Oui

Exemple — Séquence

Calculer la précondition de [x := x+1 ; y := x*2](y > 8) :

  • Étape 1 — wp de la dernière instruction : wp(y := x*2, y > 8)
    Substituer y ← x*2 dans y > 8 → x*2 > 8 → x > 4
  • Étape 2 — wp de la première instruction : wp(x := x+1, x > 4)
    Substituer x ← x+1 dans x > 4 → x+1 > 4 → x > 3

Résultat : x > 3. Aucun invariant, aucun variant — calcul direct, terminé.

Exemple — Condition if/else

Calculer la précondition de if x > 0 then y := x else y := -x end avec postcondition y ≥ 0 :

Formule : wp(if B then S₁ else S₂, Q) = (B ⊃ wp(S₁, Q)) ∧ (¬B ⊃ wp(S₂, Q))

  • Branche alors (x > 0) : wp(y := x, y ≥ 0) → x ≥ 0
    Obligation : x > 0 ⊃ x ≥ 0 → vraie (si x > 0 alors x ≥ 0). ✓
  • Branche sinon (x ≤ 0) : wp(y := -x, y ≥ 0) → -x ≥ 0 → x ≤ 0
    Obligation : x ≤ 0 ⊃ x ≤ 0 → triviale. ✓

Résultat : wp = true — la postcondition est toujours satisfaite, quelle que soit la valeur de x. Aucun invariant, aucun variant. Les deux branches ont été traitées séparément et la conjonction est vraie.

Conclusion : dès que tu vois un while, sors l'invariant et le variant. Pour tout le reste, remonte la wp mécaniquement.

L'invariant — intuition

Un invariant de boucle est une propriété sur les variables qui est vraie avant d'entrer dans la boucle, après chaque tour, et donc encore vraie quand on en sort.

Analogie : imagine un arbitre qui surveille une partie d'échecs. Il ne vérifie pas chaque case à chaque microseconde — il vérifie juste une règle simple à chaque coup : "le roi noir n'est pas en échec illégal". Si cette règle est vraie au départ et reste vraie à chaque coup, elle est vraie à la fin. C'est exactement le principe de l'invariant.

Pour ce programme, l'invariant est : total + x × y = a × b.
Traduction : à tout moment, ce qu'on a déjà accumulé dans total + ce qu'il reste à traiter (x × y) = le résultat final voulu.

Comme une caisse enregistreuse dont le sous-total + les articles non encore scannés = le total final — à tout instant de la caisse.

Trace concrète — a = 6, b = 3 (résultat attendu : 18)

L'invariant est vrai à chaque ligne — c'est ça la conservation. Le variant passe de 6 → 3 → 1 → 0 : il décroît strictement, donc la boucle s'arrête.

La notation [S]P — c'est quoi ?

[S]P se lit : "après avoir exécuté S, P est vrai". C'est la notation wp (weakest precondition) appliquée au programme S avec postcondition P.

On la calcule en remontant les substitutions à l'envers, comme on l'a fait en exo 1 :

  • Pour une affectation x := e : [x := e]P = P dans lequel on remplace x par e.
  • Pour une séquence S1 ; S2 : on commence par S2, puis on applique S1 au résultat.
  • Pour un if B then S1 else S2 : on traite les deux branches séparément.

Exemple rapide :

  • [x := x+1](x > 5) = x+1 > 5 = x > 4
  • [x := x/2 ; y := y*2](total + x×y = K) = substituer d'abord y ← y×2, puis x ← x/2 dans le prédicat.

Pourquoi cette notation ? Parce que le prof exprime la conservation de l'invariant directement en termes de wp : dire "I est vrai après un tour" c'est dire que I est la postcondition du corps S — soit [S]I. Vérifier que I ∧ E ⊃ [S]I, c'est calculer [S]I et montrer que I ∧ E l'implique.

Les 4 obligations de preuve du prof — expliquées

Pour prouver qu'une boucle while E do S end est totalement correcte (résultat juste et terminaison garantie), le prof décompose la preuve en 4 obligations. Voici ce que chacune signifie concrètement.

1. Invariance : (I ∧ E) ⊃ [S]I

Traduction : "si l'invariant est vrai ET qu'on est encore dans la boucle (E = x > 0), alors après avoir exécuté un tour complet du corps (S), l'invariant est encore vrai."
C'est la conservation du cours, écrite en wp. On calcule [S]I, on simplifie, et on vérifie que I∧E l'implique. Ici il faut traiter deux cas : x pair et x impair, parce que le corps contient un if.

2. Typage : (I ∧ E) ⊃ V ∈ ℕ

Traduction : "pendant que la boucle tourne, le variant est bien un entier naturel (≥ 0)."
C'est une condition préalable pour pouvoir utiliser V comme compteur de terminaison : un compteur qui descend doit rester positif pour qu'on sache qu'il va atteindre 0. Ici c'est immédiat car I contient déjà x ∈ ℕ et V = x.

3. Terminaison : (I ∧ E ∧ V = δ) ⊃ [S](V < δ)

Traduction : "si le variant vaut δ au début d'un tour, alors après ce tour il est strictement inférieur à δ."
δ (delta) est une variable fantôme qui "photographie" la valeur de V avant le tour. C'est comme un arbitre qui note le score avant la manche, pour vérifier qu'il a baissé après. On calcule [S](V < δ) et on vérifie que I∧E∧(V=δ) l'implique. Comme pour l'invariance, deux cas à traiter (x pair / x impair).

4. Finalisation : (I ∧ ¬E) ⊃ R

Traduction : "quand la boucle s'arrête (¬E = x ≤ 0), l'invariant implique la postcondition R."
C'est la conclusion du cours. Ici : I contient x ∈ ℕ et ¬E donne x ≤ 0, donc x = 0. L'invariant devient alors total + 0×y = a×b, soit total = a×b. ✓

Correction partielle vs. correction totale — ce que ça change concrètement

Correction partielle

La promesse conditionnelle : Si le programme s'arrête un jour, alors il donnera à 100% le bon résultat.

Analogie : c'est le livreur qui promet "si j'arrive chez toi, ce sera bien la bonne commande" — mais qui ne garantit absolument pas qu'il ne va pas tourner en rond indéfiniment sur un rond-point sans jamais trouver ta maison.

On sait que le résultat sera juste. On ne sait pas que la boucle s'arrêtera.

Correction totale

La promesse inconditionnelle : le programme s'arrête toujours et donne toujours le bon résultat.

Analogie : le livreur garantit à la fois qu'il arrivera chez toi et qu'il aura la bonne commande. Pas de round-point infini, pas de mauvaise surprise.

Correction totale = Correction partielle + Terminaison. C'est le combo ultime.

Le lien avec l'invariant et le variant :

À l'examen — ce que ça change selon l'énoncé

Si l'examen demande de prouver uniquement la correction partielle, tu peux ignorer tout ce qui concerne le variant. Il te suffit de valider 3 points sur les 4 obligations + l'établissement :

  • Établissement — I est vraie avant d'entrer dans la boucle (pas de numéro dans les 4 obligations wp du prof, mais obligatoire)
  • 1. Invariance : (I ∧ E) ⊃ [S]I
  • 4. Finalisation : (I ∧ ¬E) ⊃ R

Si l'examen demande de prouver la correction totale (ou la terminaison en plus), il faut ajouter :

  • 2. Typage : (I ∧ E) ⊃ V ∈ ℕ
  • 3. Terminaison : (I ∧ E ∧ V = δ) ⊃ [S](V < δ)
Q1 4 pts Intermédiaire

En utilisant l'invariant I ≡ x ∈ ℕ ∧ total + x × y = a × b, vérifiez les trois propriétés : établissement, conservation, conclusion.

Piège courant : pour la conservation, oublier de traiter les deux cas séparément. Le corps de la boucle se comporte différemment selon que x est pair ou impair — il faut analyser chaque branche du if.
Avant la boucle : x = a, y = b, total = 0. Calculer total + x × y dans cet état. Est-ce bien égal à a × b ? Et x ∈ ℕ est-il satisfait ?
Supposer I vraie et x > 0. Traiter deux cas :
• x impair : le corps exécute total += y, puis x := x−1, puis x := x/2, puis y := y*2. Calculer total' + x' × y' et vérifier que cela vaut toujours a × b.
• x pair : le corps saute le if, puis x := x/2, y := y*2. Même vérification.

Solution — Q1

Invariant : I ≡ x ∈ ℕ ∧ total + x × y = a × b  ·  E : x > 0

Version prof — notation wp (Invariance + Finalisation)

1. Invariance : (I ∧ E) ⊃ [S]I

Traduction : "si l'invariant est vrai ET la boucle continue, alors après avoir exécuté le corps S, l'invariant est encore vrai." On calcule [S]I avec le wp, puis on montre que I∧E l'implique.

  • x impair : on calcule [total:=total+y ; x:=x−1 ; x:=x/2 ; y:=y×2](total + x×y = a×b)
  • → en remontant les substitutions : total + y + (x−1)/2 × 2y = a×b
  • → simplifié : total + y + (x−1)y = a×b → total + xy = a×b
  • → c'est exactement la partie de I sur total, donc I∧E l'implique. ✓
  • x pair : on calcule [x:=x/2 ; y:=y×2](total + x×y = a×b)
  • → substitution : total + (x/2)×2y = a×b → total + xy = a×b. ✓

4. Finalisation : (I ∧ ¬E) ⊃ R

Traduction : "quand la boucle s'arrête (¬E = x ≤ 0), l'invariant implique la postcondition R." Comme I contient x ∈ ℕ et ¬E donne x ≤ 0, on conclut x = 0.

  • x ∈ ℕ ∧ x ≤ 0 → x = 0.
  • total + 0×y = a×b → total = a×b. ✓
Q2 2 pts Fondamental

En utilisant le variant V = x, prouvez la terminaison du programme : montrez que V est positif pendant la boucle et qu'il décroît strictement à chaque itération.

Piège courant : vérifier la décroissance uniquement pour le cas pair. Il faut traiter les deux branches du if et montrer que dans chaque cas, x diminue strictement.
La condition de boucle est x > 0, donc V = x ≥ 1 pendant l'exécution. Pour la décroissance : après le corps, x vaut (x−1)/2 si impair ou x/2 si pair. Comparer ces valeurs à x dans chaque cas.

Solution — Q2

Variant : V = x

Version prof — notation wp (Typage + Terminaison)

2. Typage : (I ∧ E) ⊃ V ∈ ℕ

Traduction : "pendant que la boucle tourne, le variant est bien un entier naturel." C'est nécessaire pour qu'il puisse servir de compteur de terminaison. Ici c'est immédiat : I contient déjà x ∈ ℕ, et V = x.

  • I ∧ E contient x ∈ ℕ, donc V = x ∈ ℕ. ✓

3. Terminaison : (I ∧ E ∧ V = δ) ⊃ [S](V < δ)

Traduction : "si le variant vaut δ avant un tour, il vaut strictement moins après." δ est une variable fantôme qui "photographie" la valeur de V au début du tour, pour pouvoir comparer avec sa valeur après le corps S. On calcule [S](x < δ) et on vérifie que I∧E∧(x=δ) l'implique.

  • x impair : [total:=total+y ; x:=x−1 ; x:=x/2 ; y:=y×2](x < δ) → (x−1)/2 < δ.
  • → I∧E∧(x=δ) donne x ≥ 1 et x=δ, donc (x−1)/2 < x = δ. ✓
  • x pair : [x:=x/2 ; y:=y×2](x < δ) → x/2 < δ.
  • → x=δ et x ≥ 2 (pair, x > 0), donc x/2 < x = δ. ✓
Bilan — correction totale
  • Invariant établi, conservé, et impliquant la postcondition à la sortie → correction partielle.
  • Variant strictement positif et décroissant → terminaison garantie.
  • Les deux ensemble : le programme est totalement correct.

Note : la convergence est logarithmique — au plus ⌊log₂ a⌋ + 1 tours, car x est au moins divisé par 2 à chaque itération.
