Exercice 1 — Calculer

Fondamental ◎ 4 pts ⏱ 10 min
Objectifs
  • Calculer des préconditions les plus faibles par application mécanique des règles.
  • Distinguer le cas wp = false (postcondition inatteignable) du cas wp = true (postcondition toujours vraie).

Prérequis : Cours — Calcul de la précondition la plus faible (wp)

Contexte — Préconditions les plus faibles

Étant donné un programme S et une postcondition Q, la précondition la plus faible wp(S, Q) est la condition initiale la moins restrictive qui garantit que S se termine en satisfaisant Q.

Si wp(S, Q) = false, aucun état initial ne permet d'atteindre Q via S. Si wp(S, Q) = true, tout état initial convient.

Règle de l'affectation :

wp(x := e, Q) = Q[x ← e]

On substitue mécaniquement toute occurrence de x dans Q par l'expression e. Si x n'apparaît pas dans Q, alors Q reste inchangée.

Règle de la composition séquentielle :

wp(S₁ ; S₂, Q) = wp(S₁, wp(S₂, Q))

On calcule de droite à gauche : d'abord la wp de l'instruction la plus à droite, puis on remonte vers la gauche en utilisant le résultat intermédiaire comme nouvelle postcondition.

Exemple

Calculer wp([x := x+3 ; x := x-1], x > 10) :

  • Étape 1 : wp(x := x-1, x > 10) = (x-1) > 10 = x > 11
  • Étape 2 : wp(x := x+3, x > 11) = (x+3) > 11 = x > 8

Résultat : x > 8. On substitue de droite à gauche, étape par étape.

Analogie

Intuition : la wp est un voyage à rebours. Pour garantir l'arrivée, on remonte le chemin instruction par instruction, en propageant la condition vers l'amont.

Comme un GPS recalculant depuis la destination : on cherche d'où partir pour être sûr d'arriver.

Q1 1 pt Fondamental

Calculer wp([x := x+2 ; x := x+1], x > 9).

Piège courant : appliquer les substitutions dans le mauvais sens (de gauche à droite). La règle de séquence exige de partir de la postcondition et de remonter de droite à gauche.
Appliquer la règle de séquence : wp(S1 ; S2, Q) = wp(S1, wp(S2, Q)). Commencer par calculer wp(x := x+1, x > 9), puis utiliser ce résultat comme postcondition de wp(x := x+2, …).

Solution — Q1

  • Étape 1 — wp(x := x+1, x > 9)
    Substituer x ← x+1 dans x > 9 → x+1 > 9 → x > 8
  • Étape 2 — wp(x := x+2, x > 8)
    Substituer x ← x+2 dans x > 8 → x+2 > 8 → x > 6

Résultat : wp([x := x+2 ; x := x+1], x > 9) = x > 6

On remonte de droite à gauche : chaque affectation décale le seuil de 1 puis de 2, ce qui ramène la contrainte initiale x > 9 à x > 6.

Q2 2 pts Intermédiaire

Calculer wp([y := x+y ; x := y-x], y ≥ 5 ∧ x ≤ 3).

Piège courant : oublier que y a changé de valeur après la première affectation. À l'étape 2, le y dans x := y-x désigne la nouvelle valeur de y (i.e. x+y initial), pas l'ancienne.
Commencer par wp(x := y-x, y ≥ 5 ∧ x ≤ 3) : substituer x ← y-x dans la postcondition. La partie y ≥ 5 est inchangée (pas de x) ; x ≤ 3 devient y-x ≤ 3, soit x ≥ y-3.
Après l'étape 1, on obtient y ≥ 5 ∧ x ≥ y-3. Appliquer maintenant wp(y := x+y, y ≥ 5 ∧ x ≥ y-3) : substituer y ← x+y. Simplifier chaque contrainte séparément.

Solution — Q2

  • Étape 1 — Substituer x ← y-x dans y ≥ 5 ∧ x ≤ 3 : y ≥ 5 ∧ (y-x) ≤ 3 → y ≥ 5 ∧ x ≥ y-3
  • Étape 2 — Substituer y ← x+y dans y ≥ 5 ∧ x ≥ y-3 :
    (x+y) ≥ 5 ∧ x ≥ (x+y)-3
    → x+y ≥ 5 ∧ x ≥ x+y-3
    → x+y ≥ 5 ∧ 0 ≥ y-3
    → x+y ≥ 5 ∧ y ≤ 3

Résultat : wp([y := x+y ; x := y-x], y ≥ 5 ∧ x ≤ 3) = x+y ≥ 5 ∧ y ≤ 3

La précondition est satisfiable : par exemple x = 3, y = 2 vérifie 3+2 ≥ 5 et 2 ≤ 3.

Q3 1 pt Fondamental

Calculer wp([x := y ; y := x²], y > x).

Piège courant : substituer x ← y dans la postcondition finale sans avoir d'abord propagé y := x². La règle de séquence impose de partir de la postcondition et de remonter instruction par instruction de droite à gauche.
Commencer par wp(y := x², y > x) : substituer y ← x² dans y > x. Quel prédicat obtient-on ?
L'étape 1 donne x² > x. Appliquer maintenant wp(x := y, x² > x) : substituer x ← y dans x² > x.

Solution — Q3

  • Étape 1 — wp(y := x², y > x)
    Substituer y ← x² dans y > x :
    = x² > x
  • Étape 2 — wp(x := y, x² > x)
    Substituer x ← y dans x² > x :
    = y² > y

Résultat : wp([x := y ; y := x²], y > x) = y² > y

Explication

La condition y² > y est équivalente à y(y−1) > 0, c'est-à-dire y < 0 ou y > 1. Le programme satisfait la postcondition exactement quand la valeur initiale de y est strictement négative ou strictement supérieure à 1.
