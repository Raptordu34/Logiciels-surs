Exercice 2 — Boucle produit

Intermédiaire ◎ 6 pts ⏱ 15 min
Objectifs
  • Déterminer l'invariant et le variant d'une boucle.
  • Vérifier les trois propriétés de l'invariant (établissement, conservation, conclusion) et la terminaison.

Prérequis : Cours — Invariants de boucle et correction de programmes

Programme à analyser

La boucle suivante calcule le produit de tous les éléments d'un tableau a de N entiers, indicé de 1 à N, avec N ≥ 0. 

prod := 1;
i    := 0;
while i < N do
    i    := i + 1;
    prod := prod * a(i);
end

Postcondition visée : prod = a(1) × a(2) × … × a(N)

Cas limite : si N = 0, la boucle ne s'exécute pas et prod = 1. C'est la valeur du produit vide — convention classique en mathématiques, analogue à la somme vide égale à 0.

C'est quoi un invariant ? (version vulgarisée)

Imagine que tu comptes des billets de banque un par un. À chaque instant, tu peux noter sur un papier : « j'ai compté i billets, et la somme dans ma main est correcte ». Cette phrase est vraie avant de commencer (0 billets comptés, somme = 0), vraie après chaque billet, et encore vraie quand tu as fini.

C'est exactement ça un invariant de boucle : une propriété sur les variables qui reste vraie à chaque tour, du début à la fin. Elle décrit ce que la boucle a accompli jusqu'ici.

Le but : quand la boucle s'arrête, l'invariant + la condition d'arrêt doivent ensemble prouver que le programme a bien fait ce qu'on voulait (la postcondition).

C'est quoi un variant ? (version vulgarisée)

Le variant répond à la question : « est-ce que la boucle va forcément s'arrêter ? »

C'est un nombre entier positif qu'on associe à la boucle, et qui diminue strictement à chaque tour. Comme un compte à rebours : 5, 4, 3, 2, 1… il ne peut pas descendre indéfiniment sous 0, donc la boucle doit s'arrêter.

Pour la boucle de cet exercice, le variant naturel est N - i : il vaut N au départ, diminue de 1 à chaque tour, et atteint 0 quand i = N (fin de boucle).

Invariant I — trois propriétés à vérifier :

  • Établissement — l'invariant est vrai avant d'entrer dans la boucle.
    Concrètement : vérifier que I est satisfait avec les valeurs d'initialisation.
  • Conservation — si I est vrai avant un tour et que la condition de boucle est vraie, alors I est encore vrai après ce tour.
    Concrètement : supposer I vrai et Cond vraie, exécuter le corps de boucle, montrer que I tient toujours.
  • Conclusion — à la sortie (I ∧ ¬Cond), on peut déduire la postcondition.
    Concrètement : la négation de la condition combinée à l'invariant doit suffire à conclure.

Variant V — deux propriétés à vérifier :

  • Positivité — tant que la boucle continue (Cond vraie), V > 0.
  • Décroissance stricte — à chaque tour, V diminue d'au moins 1.

Repère

L'invariant doit être assez fort pour impliquer la postcondition à la sortie, et assez faible pour être vrai dès l'initialisation. Si l'invariant est trop fort, il ne sera pas établi ; s'il est trop faible, il n'impliquera pas la postcondition.

Exemple concret — trace pas à pas (N = 3, a = [2, 5, 4])

Regardons la boucle produit tourner sur un petit tableau :

Exemple parallèle — boucle somme

Même structure, mais avec une addition :

sum := 0; i := 0;
while i < N do
    i   := i + 1;
    sum := sum + a(i);
end
  • Invariant : sum = a(1)+…+a(i) ∧ 0 ≤ i ≤ N
  • Variant : N - i

La boucle produit est identique en structure : remplacer + par × et la valeur neutre 0 par 1 (car le produit vide = 1, comme la somme vide = 0).

Q1 6 pts Intermédiaire

Donner l'invariant et le variant de la boucle permettant d'établir la postcondition prod = a(1) × a(2) × … × a(N).

Vérifier ensuite les trois propriétés de l'invariant (établissement, conservation, conclusion) et la terminaison.

Piège courant : écrire un invariant trop fort (prod = a(1)×…×a(N) ∧ i = N) ou trop faible (i ≤ N seul). L'invariant doit progresser avec la boucle : quand i est partiel, le produit doit être partiel. La contrainte 0 ≤ i ≤ N est indispensable pour assurer l'établissement et la conclusion.
L'invariant doit relier prod à i à chaque instant. À l'initialisation (i = 0, prod = 1), quel est le produit partiel a(1) × … × a(0) (produit des 0 premiers éléments) ? Rappel : le produit vide vaut 1 par convention.
L'invariant s'écrit : prod = a(1) × … × a(i) ∧ 0 ≤ i ≤ N. Vérifier maintenant la conservation : si cette propriété est vraie avant le corps et que i < N, est-elle encore vraie après i := i+1 ; prod := prod * a(i) ? Attention à l'ordre : i est incrémenté en premier.
Pour la conclusion : à la sortie de boucle, la condition est i ≥ N, et l'invariant impose i ≤ N, donc i = N à la sortie. Pour le variant : quel entier est strictement positif quand i < N et décroît exactement de 1 à chaque tour ?

Solution — Boucle produit

Invariant :

I : prod = a(1) × a(2) × … × a(i) ∧ 0 ≤ i ≤ N

Lecture : prod contient à tout moment le produit des i premiers éléments du tableau. Quand i = 0, le produit est vide et vaut 1. La contrainte 0 ≤ i ≤ N délimite le domaine de la variable de comptage.

Variant :

V = N - i

Vérification — Établissement :

  • Avant la boucle : i = 0 et prod = 1.
  • prod = 1 = a(1)×…×a(0) (produit vide = 1) ✓
  • 0 ≤ 0 ≤ N (car N ≥ 0 par hypothèse) ✓

Vérification — Conservation :

  • Hypothèse : prod = a(1)×…×a(i) ∧ 0 ≤ i ≤ N et i < N.
  • Après i := i+1 : le nouveau i vaut i+1, avec i+1 ≤ N.
  • Après prod := prod * a(i+1) : prod = a(1)×…×a(i) × a(i+1) = a(1)×…×a(i+1) ✓
  • 0 ≤ i+1 ≤ N (car i < N implique i+1 ≤ N) ✓

Vérification — Conclusion :

  • À la sortie : I ∧ ¬(i < N) ⇒ prod = a(1)×…×a(i) ∧ i ≥ N ∧ i ≤ N
  • Donc i = N, et prod = a(1) × a(2) × … × a(N) ✓

Vérification — Terminaison (variant) :

  • Tant que i < N : V = N-i ≥ 1 > 0 ✓
  • Après chaque itération : V passe de N-i à N-(i+1) = V-1, strictement décroissant ✓
Bilan
  • Invariant établi, conservé, et impliquant la postcondition à la sortie → correction partielle.
  • Variant strictement positif et décroissant → terminaison garantie.
  • Les deux ensemble établissent la correction totale du programme.
