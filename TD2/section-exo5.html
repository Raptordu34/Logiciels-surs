Exercice 5 — Feux de circulation

Avancé ◎ 8 pts ⏱ 25 min
Objectifs
  • Modéliser un système réel en machine abstraite B.
  • Distinguer propriété de séquentialité et propriété de sûreté.
  • Écrire des opérations avec préconditions qui garantissent l'invariant.

Prérequis : Machines abstraites B, variables, invariant, opérations, préconditions, fonctions totales

Contexte — Carrefour à deux directions

On modélise un carrefour à 4 voies contrôlé par des feux dans deux directions : NordSud et EstOuest. Les feux Est et Ouest sont toujours de la même couleur, idem pour Nord et Sud — on les traite donc comme une seule entité par direction.

Les feux peuvent prendre trois couleurs : Rouge, Vert, Orange.

Propriété de séquentialité

La seule suite de couleurs autorisée est :

Rouge → Vert → Orange → Rouge → …

  • Seul un feu Orange peut passer au Rouge.
  • Seul un feu Rouge peut passer au Vert.
  • Seul un feu Vert peut passer à l'Orange.

Propriété de sûreté

Les deux directions ne peuvent pas circuler en même temps :

Si une direction est Vert ou Orange, l'autre doit être Rouge.

En d'autres termes : au moins une des deux directions est toujours Rouge. Sans cette propriété, deux voitures pourraient traverser le carrefour simultanément — collision assurée.

Il y a 3 opérations à spécifier, chacune prenant une direction en paramètre :

  • PasseAuRouge(dir) — met le feu de dir au Rouge.
  • PasseAuVert(dir) — met le feu de dir au Vert.
  • PasseAlOrange(dir) — met le feu de dir à l'Orange.
Question Q5.1 : écrire la Machine Abstraite B complète.

C'est quoi une Machine Abstraite B ?

Une Machine Abstraite B, c'est une façon formelle de décrire un système avec des variables d'état et des opérations qui les modifient. L'idée centrale : on définit un invariant — une propriété qui doit rester vraie en permanence — et on prouve que chaque opération le maintient.

Analogie : pense à un compte bancaire. L'invariant serait "le solde est ≥ 0". L'opération "retirer X€" a une précondition "solde ≥ X" pour garantir que l'invariant reste vrai. La machine ne gère pas le cas interdit — c'est à l'appelant de ne pas appeler l'opération hors conditions.

Structure d'une machine :

MACHINE NomDeLaMachine
SETS        -- les types/ensembles de valeurs possibles
VARIABLES   -- les variables d'état
INVARIANT   -- propriété permanente sur les variables
INITIALISATION  -- valeurs initiales (doivent établir l'invariant)
OPERATIONS  -- les opérations modifiant l'état
END

La notation feux ∈ DIRECTION → FEU — c'est quoi ?

DIRECTION → FEU désigne l'ensemble des fonctions totales de DIRECTION vers FEU. Dire que feux ∈ DIRECTION → FEU, c'est dire que feux est une fonction qui associe à chaque direction exactement une couleur.

Concrètement :

  • feux(NordSud) = couleur du feu NordSud (ex : Rouge)
  • feux(EstOuest) = couleur du feu EstOuest (ex : Vert)

C'est comme un tableau à deux cases. Modifier feux(dir) := Rouge revient à changer la valeur à l'index dir dans ce tableau.

En B, {NordSud ↦ Rouge, EstOuest ↦ Rouge} est la notation pour la fonction qui envoie NordSud sur Rouge et EstOuest sur Rouge — c'est la valeur initiale de feux.

Comprendre l'invariant — deux formes équivalentes

La propriété de sûreté ("au moins une direction est Rouge") peut s'écrire de deux façons logiquement équivalentes :

Forme 1 — disjonction directe

feux(NordSud) = Rouge
∨ feux(EstOuest) = Rouge

"Au moins une des deux est Rouge." Simple et direct, mais moins explicite sur ce qui se passe quand une est Verte.

Forme 2 — implications (version prof Allel)

(feux(NordSud) ∈ {Vert,Orange}
  ⟹ feux(EstOuest) = Rouge)
∧
(feux(EstOuest) ∈ {Vert,Orange}
  ⟹ feux(NordSud) = Rouge)

"Si NordSud circule, EstOuest est Rouge — et vice versa." Plus lisible en termes de sûreté réelle.

Ces deux formes sont logiquement équivalentes grâce à la règle p ⟹ q ≡ ¬p ∨ q. La forme 1 est plus compacte ; la forme 2 est plus proche du langage naturel de la propriété de sûreté.

Pourquoi ces préconditions pour chaque opération ?

Chaque opération doit garantir que l'invariant reste vrai après son exécution. La précondition est choisie exactement pour ça.

OpérationPREPourquoi ?
PasseAuRouge(dir) feux(dir) = Orange Séquentialité : seul Orange → Rouge est légal.
Sûreté : si dir est Orange, l'autre est déjà Rouge (invariant). Passer dir au Rouge ne change rien à la sûreté. Pas besoin de condition supplémentaire.
PasseAuVert(dir) feux(dir) = Rouge
∧ autre dir = Rouge 		Séquentialité : seul Rouge → Vert est légal.
Sûreté : après, dir sera Vert → l'autre DOIT être Rouge. On l'exige explicitement dans la PRE, car les deux pourraient être Rouges au départ sans que l'invariant l'impose.
PasseAlOrange(dir) feux(dir) = Vert Séquentialité : seul Vert → Orange est légal.
Sûreté : si dir est Vert, l'autre est déjà Rouge (invariant). Passer à Orange maintient "au moins une est Rouge". Pas besoin de condition supplémentaire.

Résumé : seule PasseAuVert a besoin d'une condition de sûreté explicite, parce que c'est la seule opération qui fait passer une direction de Rouge (inoffensif) à Vert (bloquant pour l'autre). Pour les deux autres, l'invariant garantit déjà la sûreté.

  • SETS — types énumérés : MON_TYPE = {val1, val2, val3}
  • INVARIANT — propriété permanente, conjonction de prédicats.
  • feux ∈ DIRECTION → FEU — feux est une fonction totale de DIRECTION dans FEU.
  • feux(dir) := val — affectation fonctionnelle : change la valeur pour la clé dir.
  • {NordSud ↦ Rouge, EstOuest ↦ Rouge} — définition en extension d'une fonction.

Structure d'une opération avec paramètre :

NomOp(param) =
    PRE
        param ∈ TYPE ∧ condition_sur_etat
    THEN
        variable := nouvelle_valeur
    END

Le paramètre param doit toujours être contraint dans la PRE (param ∈ TYPE) pour que la machine soit bien typée.

Q5.1 8 pts Avancé

Écrire la Machine Abstraite B complète FeuxTricolores modélisant le carrefour : sets, variables, invariant, initialisation, et les trois opérations avec leurs préconditions.

Piège courant : oublier la condition de sûreté dans la PRE de PasseAuVert. La condition séquentielle feux(dir) = Rouge seule ne suffit pas : les deux feux pourraient être Rouges, et passer l'un au Vert sans vérifier l'autre violerait l'invariant si on n'y prend garde.
L'invariant comporte deux parties :
1. Le typage : feux ∈ DIRECTION → FEU (chaque direction a une couleur).
2. La sûreté : au moins une direction est Rouge — soit feux(NordSud) = Rouge ∨ feux(EstOuest) = Rouge.
Pour PasseAuVert(dir), la PRE doit contenir trois conditions :
• dir ∈ DIRECTION (typage du paramètre)
• feux(dir) = Rouge (séquentialité : Rouge → Vert)
• l'autre direction est Rouge (sûreté), exprimée par une implication selon dir.

Solution — Q5.1

MACHINE FeuxTricolores

SETS
    DIRECTION = {NordSud, EstOuest};
    FEU = {Rouge, Vert, Orange}

VARIABLES
    feux

INVARIANT
    feux ∈ DIRECTION → FEU
    ∧
    (feux(NordSud) = Rouge ∨ feux(EstOuest) = Rouge)

INITIALISATION
    feux := {NordSud ↦ Rouge, EstOuest ↦ Rouge}

OPERATIONS

    PasseAuRouge(dir) =
    PRE
        -- séquentialité : seul Orange peut passer au Rouge
        dir ∈ DIRECTION ∧ feux(dir) = Orange
    THEN
        feux(dir) := Rouge
    END
    ;

    PasseAuVert(dir) =
    PRE
        -- séquentialité : seul Rouge peut passer au Vert
        dir ∈ DIRECTION ∧ feux(dir) = Rouge
        ∧
        -- sûreté : la direction opposée doit être Rouge
        (dir = NordSud  ⟹ feux(EstOuest) = Rouge)
        ∧
        (dir = EstOuest ⟹ feux(NordSud)  = Rouge)
    THEN
        feux(dir) := Vert
    END
    ;

    PasseAlOrange(dir) =
    PRE
        -- séquentialité : seul Vert peut passer à l'Orange
        dir ∈ DIRECTION ∧ feux(dir) = Vert
    THEN
        feux(dir) := Orange
    END

END
Vérification — l'initialisation établit l'invariant
  • feux = {NordSud ↦ Rouge, EstOuest ↦ Rouge} est bien dans DIRECTION → FEU. ✓
  • feux(NordSud) = Rouge → la disjonction est vraie. ✓
Vérification — chaque opération maintient l'invariant
  • PasseAuRouge(dir) : PRE impose feux(dir) = Orange. Par l'invariant, l'autre direction est déjà Rouge. Après : feux(dir) = Rouge → la disjonction reste vraie. ✓
  • PasseAuVert(dir) : PRE impose que l'autre direction est Rouge. Après : feux(dir) = Vert, l'autre = Rouge → disjonction vraie. ✓
  • PasseAlOrange(dir) : PRE impose feux(dir) = Vert. Par l'invariant, l'autre est Rouge. Après : feux(dir) = Orange, l'autre = Rouge → disjonction vraie. ✓

Variante — invariant sous forme d'implications (Allel)

L'invariant peut aussi s'écrire avec des implications — logiquement équivalent :

INVARIANT
    feux ∈ DIRECTION → FEU
    ∧
    (feux(NordSud)  ∈ {Vert, Orange} ⟹ feux(EstOuest) = Rouge)
    ∧
    (feux(EstOuest) ∈ {Vert, Orange} ⟹ feux(NordSud)  = Rouge)

Équivalence : feux(NordSud) = Rouge ∨ feux(EstOuest) = Rouge est la même chose que (feux(NordSud) ≠ Rouge ⟹ feux(EstOuest) = Rouge), et feux(NordSud) ≠ Rouge ≡ feux(NordSud) ∈ {Vert, Orange}. La version avec implications est plus explicite sur la contrainte de sûreté.
