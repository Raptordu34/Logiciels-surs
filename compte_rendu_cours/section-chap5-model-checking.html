Chap 5 — Model-Checking & Logiques Temporelles

Synthèse

  • Systèmes réactifs — interagissent continuellement avec leur environnement (sans fin définie), justifiant l'utilisation d'automates d'états.
  • Model-Checking — technique de vérification exhaustive explorant tous les états d'un modèle pour s'assurer qu'il respecte une propriété.
  • LTL (Linéaire) — vérifie des séquences linéaires d'états avec des opérateurs : $\bigcirc$ (Next), $\lozenge$ (Future/F), $\square$ (Always/G), $U$ (Until).
  • CTL (Arborescente) — vérifie des arbres de calculs via des quantificateurs : $A$ (Tous les chemins) ou $E$ (Au moins un chemin), combinés aux opérateurs temporels.
  • Relation de satisfaction — notée $M \models \varphi$, indique si le modèle $M$ respecte la formule $\varphi$.

1. Systèmes réactifs et propriétés temporelles

  • Prend une entrée, calcule, et s'arrête (terminaison).
  • Données complexes, contrôle simple.
  • Vérifié par triples de Hoare (Sémantique axiomatique).
  • Ne se termine pas, maintient une interaction (OS, protocoles).
  • Données simples, contrôle complexe (concurrence).
  • Nécessite de vérifier des propriétés dans le temps (Model-Checking).

Les propriétés temporelles que l'on souhaite vérifier sont :

  • Sûreté (Safety) : Quelque chose de mauvais n'arrive jamais (ex: pas de division par 0).
  • Vivacité (Liveness) : Quelque chose de bon finit par arriver (ex: le message finit par être transmis).
  • Accessibilité : Un certain état peut être atteint.

2. Systèmes de transitions (Automates)

Un système de transitions (ST) modélise le comportement du système. Il est défini par un quadruplet $S = (Q, E, T, q_0)$ :

  • $Q$ : ensemble fini d'états.
  • $E$ : ensemble fini d'étiquettes de transitions.
  • $T \subseteq Q \times E \times Q$ : ensemble de transitions.
  • $q_0$ : état initial.

Pour le Model-Checking, on l'étend en Structure de Kripke avec une fonction d'étiquetage qui indique quelles propositions atomiques sont vraies dans chaque état.

3. Logiques Temporelles (LTL, CTL)

Les logiques temporelles ajoutent à la logique propositionnelle des opérateurs permettant de parler du futur.

LTL (Logique Temporelle Linéaire)

En LTL, le temps est vu comme une séquence linéaire (un chemin $\sigma$). Les opérateurs temporels de base sont :

Symbole Nom Signification
$\bigcirc$ ou $\mathbf{X}$NextVrai à l'instant (ou état) suivant.
$\lozenge$ ou $\mathbf{F}$FutureFinit par être vrai dans le futur.
$\square$ ou $\mathbf{G}$Globally / AlwaysToujours vrai, à partir de maintenant.
$\mathbf{U}$Until$p \mathbf{U} q$ : $p$ reste vrai jusqu'à ce que $q$ devienne vrai.
$$\square(p \implies \lozenge q)$$ Exemple LTL : Si $p$ est vrai, alors $q$ le sera dans le futur (Vivacité)

CTL (Computation Tree Logic - Logique Arborescente)

En CTL, le temps est branchant : à partir d'un état, plusieurs futurs sont possibles. On introduit les quantificateurs de chemins :

  • $\mathbf{A}$ (All) : Sur tous les chemins d'exécution partant de l'état courant.
  • $\mathbf{E}$ (Exists) : Il existe au moins un chemin d'exécution partant de l'état courant.
Formule Sémantique
$\mathbf{AG} \; p$$p$ est toujours vrai sur tous les chemins (Invariant fort).
$\mathbf{AF} \; p$Sur tous les chemins, on finit toujours par atteindre $p$.
$\mathbf{EF} \; p$Il existe au moins un chemin qui atteint $p$ (Accessibilité).
$\mathbf{EG} \; p$Il existe au moins un chemin où $p$ reste toujours vrai.
CTL* : C'est la logique la plus expressive qui combine librement quantificateurs de chemins ($A, E$) et opérateurs temporels ($X, F, G, U$). LTL et CTL sont des sous-ensembles de CTL* qui ne se recouvrent pas complètement.

4. Relation de satisfaction ($M \models \varphi$)

La question centrale du Model-Checking est : Est-ce que le modèle $M$ valide (satisfait) la formule $\varphi$ ?

$M \models \varphi$
Signifie que la formule $\varphi$ est vraie pour toutes les exécutions (ou dans l'état initial) de la structure de Kripke $M$.

Si la réponse est Non ($M \not\models \varphi$), l'outil de Model-Checking fournit un contre-exemple (une trace d'exécution menant à l'erreur), ce qui est extrêmement précieux pour le débogage.
