Exercice 7 — Calcul de wp sur une séquence

Intermédiaire ◎ 5 pts ⏱ 10 min
Objectifs
  • Appliquer la règle de séquence : calculer wp de droite à gauche sur plusieurs instructions
  • Identifier et nommer les conditions intermédiaires entre chaque instruction
  • Simplifier algébriquement une condition après substitution

Prérequis : Exercice 5 — Préconditions et Postconditions · Exercice 6 — Preuves de triplets

wp sur une séquence d'instructions

Jusqu'ici, on a calculé wp pour une seule affectation. Les programmes réels enchaînent plusieurs instructions. La règle de séquence étend wp à ce cas en traitant les instructions de droite à gauche — on commence par la dernière et on remonte vers la première.

Règle wp pour la séquence

wp(C₁ ; C₂, Q) = wp(C₁, wp(C₂, Q))

On applique wp de l'intérieur vers l'extérieur, c'est-à-dire de la dernière instruction vers la première :

  1. Calculer Q₁ = wp(C₂, Q) — la condition intermédiaire entre C₁ et C₂.
  2. Calculer wp(C₁, Q₁) — la précondition finale du programme entier.

Pour une séquence plus longue C₁ ; C₂ ; C₃, on applique la même logique :

Q₂ = wp(C₃, Q)
Q₁ = wp(C₂, Q₂)
wp = wp(C₁, Q₁)

Analogie — La chaîne de garanties

Imaginez une chaîne de montage : chaque poste de travail (C₁, C₂…) reçoit une pièce et la transforme. Pour garantir que la pièce finale respecte la norme Q, on demande au dernier poste quelles conditions il exige sur sa pièce entrante — c'est wp(C₂, Q). On remonte ensuite vers le poste précédent, et ainsi de suite jusqu'au début de la chaîne. Chaque condition intermédiaire est une "norme interne" entre deux postes.

Visualisation avec un tableau annoté

La méthode de calcul correspond à insérer des assertions logiques entre chaque instruction — comme des étiquettes sur un tapis roulant :

{ wp(C₁, Q₁) }
C₁
{ Q₁ = wp(C₂, Q) }
C₂
{ Q }

Chaque paire { assertion } instruction { assertion } forme un triplet de Hoare valide par construction.

Piège classique : calculer wp de gauche à droite (dans l'ordre d'exécution). La règle est l'inverse : on part de la postcondition et on remonte instruction par instruction vers le début. Appliquer wp dans le sens de l'exécution donne un résultat faux.

Exemple guidé

Calculer wp(x := x+1 ; y := x+y,   y > 10).

  1. Étape 1 — wp de la 2ᵉ instruction :
    wp(y := x+y,   y > 10)
    → substituer x+y à y : x+y > 10
  2. Étape 2 — wp de la 1ʳᵉ instruction sur Q₁ :
    wp(x := x+1,   x+y > 10)
    → substituer x+1 à x : (x+1)+y > 10, soit x+y > 9

Résultat : wp = x + y > 9.

Méthode générale

  • Identifier les instructions dans l'ordre (C₁ ; C₂ ; …)
  • Partir de la postcondition Q
  • Appliquer wp sur la dernière instruction → obtenir Q₁
  • Appliquer wp sur l'avant-dernière avec Q₁ → obtenir Q₀
  • Répéter jusqu'à la première instruction
  • Simplifier algébriquement le résultat final si possible
Q1 5 pts Fondamental

Soit le programme :

S  ::  x := 2*y   ;   y := y − 1

Calculer wp(S,   x = y + 5).

Piège courant : appliquer wp sur x := 2*y en premier. Il faut commencer par la dernière instruction (y := y−1) et remonter. Commencer à l'envers donne une condition intermédiaire erronée et un résultat faux.
Étape 1 : Calculez wp(y := y−1,   x = y+5). La variable assignée est y, l'expression est y−1. Substituez y−1 à la place de y dans x = y+5. Quelle condition intermédiaire Q₁ obtenez-vous ?
Q₁ = x = (y−1)+5 = x = y+4. Étape 2 : Calculez maintenant wp(x := 2*y,   x = y+4). Substituez 2*y à x dans x = y+4, puis simplifiez.

Solution — Q1

Étape 1 — wp de la 2ᵉ instruction :

wp(y := y−1,   x = y+5)
=   (x = y+5)[y−1 / y]
=   x = (y−1)+5
=   x = y+4     ← condition intermédiaire Q₁

Étape 2 — wp de la 1ʳᵉ instruction avec Q₁ :

wp(x := 2*y,   x = y+4)
=   (x = y+4)[2*y / x]
=   2*y = y+4
=   y = 4     ← précondition finale (après simplification)

Résultat : wp(S, x = y+5) = (y = 4)

Programme annoté complet
{ y = 4 }
x := 2*y
{ x = y+4 }
y := y−1
{ x = y+5 }

Chaque accolade est une assertion valide. On peut vérifier : si y = 4 au départ, alors après x := 2*y, x = 8. Ensuite après y := y−1, y = 3. La postcondition donne : 8 = 3+5 = 8. ✓

Interprétation sémantique

Le résultat y = 4 signifie que la postcondition x = y+5 n'est garantie que si y vaut exactement 4 avant l'exécution du programme. Pour toute autre valeur initiale de y, il existera un état final où x ≠ y+5.

C'est une précondition très forte (elle fixe y à une seule valeur) — ce qui reflète que la postcondition est elle-même très précise.

2 Étape 1 — wp de la 2ᵉ instruction wp(y := y−1, x=y+5) calculé correctement → x = y+4 2 Étape 2 — wp de la 1ʳᵉ instruction wp(x := 2*y, x=y+4) calculé correctement → 2y = y+4 1 Simplification et conclusion 2y = y+4 simplifié en y = 4, résultat final clairement annoncé
