Exercice 8 — Machine Abstraite (Méthode B)

Avancé ◎ 15 pts ⏱ 25 min
Objectifs
  • Analyser une machine abstraite.
  • Vérifier la préservation d'un invariant.
  • Tracer l'évolution de l'état suite à des opérations.

Contexte

Soit la machine abstraite suivante :

MACHINE q2
VARIABLES v1
INVARIANT v1 ∈ 0..4
INITIALISATION v1 := 0

OPERATIONS
  res <-- op1 =
  PRE v1 <= 2 THEN
    ANY x1 WHERE x1 ∈ NAT & x1 ∈ 0..2 THEN 
      res := x1 || v1 := v1 + x1 
    END
  END;

  res <-- op2 =
  PRE v1 ∈ NAT THEN
    ANY x1 WHERE x1 ∈ NAT & x1 ∈ 0..2 THEN 
      res := x1 || v1 := v1 + x1 
    END
  END;

  op3 =
  PRE v1 ∈ 0..2 THEN
    SELECT v1 = 0 THEN v1 := v1 + 1
    WHEN v1 = 1 THEN v1 := 0
    WHEN v1 = 2 THEN v1 := 1
    ELSE v1 := 3 END
  END
END
Q1 8 pts

(a) Indiquer si l'invariant est préservé. Justifier votre réponse.

Piège courant : Il ne suffit pas de vérifier l'invariant pour l'état initial, il faut s'assurer qu'aucune opération ne peut amener la machine dans un état violant l'invariant (v1 ∈ 0..4).
Regardez les préconditions et le comportement de chaque opération. Par exemple, pour op2, v1 peut valoir n'importe quel naturel selon la précondition, mais est-ce que ça préserve l'invariant ?
Dans op2, si v1 = 4 (ce qui respecte l'invariant), et qu'on choisit x1 = 2, alors la nouvelle valeur de v1 sera 4 + 2 = 6.

Solution — Q1

L'invariant n'est pas préservé.

L'opération op2 a pour précondition v1 ∈ NAT, qui est très large et n'empêche pas v1 de valoir 4 (ce qui est valide selon l'invariant). Or, op2 choisit un entier x1 ∈ 0..2 et fait v1 := v1 + x1.

Si la machine est dans un état où v1 = 4 (état valide), l'appel de op2 peut choisir x1 = 2. La nouvelle valeur de v1 devient alors 4 + 2 = 6.
La valeur 6 n'appartient pas à l'intervalle 0..4 défini par l'invariant. L'invariant est donc violé.

Remarque : op1 préserve l'invariant car sa précondition exige v1 <= 2, et x1 vaut au maximum 2, donc v1 + x1 <= 4.

Q2 7 pts

(b) Quel est l'état de la machine après avoir exécuté les 3 opérations suivantes :
1 <-- op1 ; 2 <-- op1 ; 2 <-- op2.

(Rappel : 1 <-- op1 signifie que l'opération a retourné la valeur 1 dans res)

Solution — Q2

L'état initial est v1 = 0.

  1. 1 <-- op1 :
    • La précondition (v1 <= 2, soit 0 <= 2) est respectée.
    • L'opération retourne res = 1, ce qui signifie que x1 = 1 a été choisi.
    • L'état devient : v1 = v1 + x1 = 0 + 1 = 1.
  2. 2 <-- op1 :
    • La précondition (1 <= 2) est respectée.
    • L'opération retourne res = 2, ce qui signifie que x1 = 2 a été choisi.
    • L'état devient : v1 = 1 + 2 = 3.
  3. 2 <-- op2 :
    • La précondition (v1 ∈ NAT) est respectée (3 est un entier naturel).
    • L'opération retourne res = 2, ce qui signifie que x1 = 2 a été choisi.
    • L'état devient : v1 = 3 + 2 = 5.

L'état final de la machine est donc v1 = 5.
