Exercice 4 — Obligations de preuve d'une boucle

Avancé ◎ 10 pts ⏱ 15–20 min
Objectifs
  • Comprendre le rôle d'un invariant de boucle dans une preuve de correction
  • Formuler les trois obligations de preuve associées à un invariant
  • Vérifier chaque obligation par calcul de wp ou implication logique
  • Distinguer correction partielle et correction totale
  • Construire un variant et prouver la terminaison d'une boucle

Prérequis : Cours 2 — Logique de Hoare · Exercice 3 (wp et validité de triplets) · Notion de variant (terminaison)

Programme à prouver

On considère la boucle suivante, où b est une constante entière quelconque :

{ x >= 0 }
while x < b do
    x := x + 1
done
{ x >= 0  ∧  ¬(x < b) }

L'invariant proposé est I : x ≥ 0. Cet exercice se déroule en deux parties :

  • Q1 — Correction partielle : vérifier les obligations Établissement, 1. Invariance et 4. Finalisation, qui établissent que si la boucle termine, alors la postcondition est satisfaite.
  • Q2 — Correction totale : trouver un variant et prouver la terminaison via les obligations 2. Typage et 3. Terminaison.

Pour prouver qu'une boucle while E do S end est correcte, on choisit un invariant I et on vérifie 4 obligations de preuve. La notation [S]P signifie : après avoir exécuté S, P est vrai — c'est la wp appliquée en remontant les substitutions.

Établissement (hors numérotation — correction partielle)

P ⊃ I

La précondition implique l'invariant : I est vrai avant d'entrer dans la boucle.

1. Invariance (correction partielle)

(I ∧ E) ⊃ [S]I

Si l'invariant est vrai et qu'on est encore dans la boucle (E), alors après un tour complet ([S]), l'invariant est encore vrai. On calcule [S]I par substitution, puis on vérifie que I ∧ E l'implique.

2. Typage (correction totale)

(I ∧ E) ⊃ V ∈ ℕ

Tant que la boucle tourne, le variant est un entier naturel (≥ 0). Sans cela, il pourrait décroître indéfiniment sans jamais atteindre zéro.

3. Terminaison (correction totale)

(I ∧ E ∧ V = δ) ⊃ [S](V < δ)

δ (delta) est une variable fantôme qui "photographie" la valeur de V juste avant un tour. Cette obligation dit : si V valait δ à l'entrée du corps, alors après le corps ([S]), V est strictement inférieur à δ. On calcule [S](V < δ) par substitution, puis on vérifie que I ∧ E ∧ V = δ l'implique.

4. Finalisation (correction partielle)

(I ∧ ¬E) ⊃ R

Quand la boucle s'arrête (¬E), l'invariant est encore vrai. Ces deux faits réunis doivent impliquer la postcondition R.

— Établissement P ⊃ I Partielle
1 Invariance (I ∧ E) ⊃ [S]I Partielle
2 Typage (I ∧ E) ⊃ V ∈ ℕ Totale
3 Terminaison (I ∧ E ∧ V = δ) ⊃ [S](V < δ) Totale
4 Finalisation (I ∧ ¬E) ⊃ R Partielle

Analogie — le compte à rebours

Le variant V comme minuteur : δ est la photo de V prise avant le tour. La terminaison exige que le minuteur ait avancé après le tour (V < δ). Comme V ∈ ℕ, il ne peut pas descendre indéfiniment — la sonnerie retentira forcément.

Exemple guidé

Boucle : { n ≥ 0 } while n > 0 do n := n − 1 done { n = 0 }, invariant I : n ≥ 0.

  • Établ. : n ≥ 0 ⊃ n ≥ 0 — trivial ✓
  • 1. Invariance : [n := n−1](n ≥ 0) = n−1 ≥ 0 = n ≥ 1. Or I ∧ E = n ≥ 0 ∧ n > 0 = n ≥ 1 ⊃ n ≥ 1 ✓
  • 4. Finalisation : (n ≥ 0 ∧ ¬(n > 0)) = (n = 0) ⊃ n = 0 ✓

Analogie

L'invariant comme règle d'entreprise : imaginez une chaîne de montage qui doit respecter la règle "aucune pièce défectueuse ne circule". L'initialisation vérifie cette règle à l'ouverture. La conservation vérifie qu'aucune opération ne l'enfreint. La postcondition vérifie que la règle suffit à garantir la qualité du produit fini.

L'invariant et le variant sont réservés aux boucles while — et uniquement elles

Les outils vus dans cet exercice — invariant I, variant V, et l'ensemble des obligations de preuve associées — sont exclusivement conçus pour les boucles while. Pourquoi cette restriction ? Parce que le while est la seule structure de programmation dont on ne sait pas à l'avance combien de fois le code va s'exécuter. La boucle peut tourner 0 fois, 3 fois, 100 fois… On ne peut donc pas dérouler mécaniquement le calcul jusqu'à la fin.

Pour toutes les autres structures — séquences et conditions if/else — le nombre d'exécutions est toujours connu. On peut remonter la précondition wp instruction par instruction, sans aucun invariant ni variant.

Structure Nb d'exécutions connu ? Méthode de preuve Invariant ? Variant ?
C₁ ; C₂ ; … ; Cₙ (séquence) Oui — 1 fois chaque instruction wp de droite à gauche, substitution mécanique Non Non
if B then C₁ else C₂ Oui — exactement une des deux branches wp des deux branches séparément + conjonction Non Non
while B do C done Non — 0, 1, ou ∞ tours possibles Invariant I + Établissement + 1.Invariance + 4.Finalisation Oui Si correction totale

Exemple — Séquence

Calculer la précondition de { ? } x := x+1 ; y := x*2 { y > 8 } :

  • Étape 1 — wp de la dernière instruction : wp(y := x*2, y > 8)
    Substituer y ← x*2 dans y > 8 → x*2 > 8 → x > 4
  • Étape 2 — wp de la première instruction : wp(x := x+1, x > 4)
    Substituer x ← x+1 dans x > 4 → x+1 > 4 → x > 3

Résultat : précondition = x > 3. Aucun invariant, aucun variant — calcul direct en 2 étapes.

Exemple — Condition if/else

Calculer la précondition de if B then C₁ else C₂ done avec postcondition Q :

Formule : wp(if B then C₁ else C₂, Q) = (B ⊃ wp(C₁, Q)) ∧ (¬B ⊃ wp(C₂, Q))

  • On calcule wp(C₁, Q) et wp(C₂, Q) séparément.
  • On combine : la branche then ne doit satisfaire sa wp que quand B est vrai, et vice versa.
  • Aucun invariant, aucun variant — deux calculs wp indépendants.

Conclusion : dès que tu vois un while, sors l'invariant. Pour tout le reste (séquences, if), remonte la wp mécaniquement.

Correction partielle vs. correction totale — ce que ça change concrètement

Correction partielle

La promesse conditionnelle : si la boucle s'arrête un jour, alors la postcondition est garantie à 100%.

Analogie : c'est le livreur qui promet "si j'arrive chez toi, ce sera bien la bonne commande" — mais qui ne garantit pas qu'il ne va pas tourner en rond indéfiniment sur un rond-point sans jamais trouver ta maison.

On sait que le résultat sera juste si ça termine. On ne sait pas si ça terminera.

Correction totale

La promesse inconditionnelle : la boucle se termine toujours et donne toujours le bon résultat.

Analogie : le livreur garantit à la fois qu'il arrivera chez toi et qu'il aura la bonne commande. Pas de rond-point infini, pas de mauvaise surprise.

Correction totale = Correction partielle + Terminaison.

À l'examen — ce que ça change selon l'énoncé

Si l'examen demande uniquement la correction partielle : tu peux ignorer entièrement le variant. Il suffit de vérifier les 3 obligations de Q1 :

  • Établissement : P ⊃ I
  • 1. Invariance : (I ∧ E) ⊃ [S]I — calculé par substitution (wp)
  • 4. Finalisation : (I ∧ ¬E) ⊃ R

Si l'examen demande la correction totale (ou la terminaison en plus), il faut ajouter les 2 obligations de Q2 :

  • 2. Typage : (I ∧ E) ⊃ V ∈ ℕ
  • 3. Terminaison : (I ∧ E ∧ V = δ) ⊃ [S](V < δ)
Q1 10 pts Avancé

Pour la boucle présentée en contexte, avec l'invariant I : x ≥ 0, formulez et vérifiez les trois obligations de preuve.

a) Établissement — montrez que P ⊃ I. (2 pts)
b) 1. Invariance — montrez que (I ∧ E) ⊃ [x := x+1]I. Calculez d'abord [x := x+1]I par substitution, puis vérifiez l'implication. (5 pts)
c) 4. Finalisation — montrez que (I ∧ ¬E) ⊃ R. (3 pts)
Piège courant : confondre l'Invariance et la Finalisation. L'Invariance utilise I ∧ E (condition vraie, on est encore dans la boucle) et calcule [S]I. La Finalisation utilise I ∧ ¬E (condition fausse, on vient d'en sortir) et implique R.
Instanciez les formules avec les valeurs du problème : P = (x ≥ 0), E = (x < b), S = (x := x+1), I = (x ≥ 0), R = (x ≥ 0 ∧ ¬(x < b)). L'établissement (a) et la finalisation (c) sont des implications directes à vérifier sans calcul de wp.
Pour l'invariance (b), calculez [x := x+1](x ≥ 0) : substituez x ← x+1 dans x ≥ 0 → x+1 ≥ 0 → x ≥ −1. Vérifiez ensuite que (I ∧ E) = (x ≥ 0 ∧ x < b) implique x ≥ −1.

Solution — Q1

Données : P = (x ≥ 0), E = (x < b), S = (x := x+1), I = (x ≥ 0), R = (x ≥ 0 ∧ ¬(x < b)).

a) Établissement — 2 pts

Formule : P ⊃ I

Instance : x ≥ 0 ⊃ x ≥ 0

Implication triviale par réflexivité. ✓

b) 1. Invariance — 5 pts

Formule : (I ∧ E) ⊃ [x := x+1]I, soit (x ≥ 0 ∧ x < b) ⊃ [x := x+1](x ≥ 0)

On calcule [x := x+1](x ≥ 0) par la règle d'affectation :

  • [x := x+1](x ≥ 0)
    = substituer x par (x+1) dans x ≥ 0
    = x+1 ≥ 0
    = x ≥ −1
  • Vérifier (x ≥ 0 ∧ x < b) ⊃ x ≥ −1
    La conjonction implique x ≥ 0, et x ≥ 0 ⊃ x ≥ −1 car 0 > −1. ✓

c) 4. Finalisation — 3 pts

Formule : (I ∧ ¬E) ⊃ R

Instance : (x ≥ 0 ∧ ¬(x < b)) ⊃ (x ≥ 0 ∧ ¬(x < b))

Implication triviale : la postcondition R est exactement I ∧ ¬E. ✓

Conclusion — Correction partielle établie

Les trois obligations sont vérifiées : la boucle est partiellement correcte vis-à-vis de l'invariant x ≥ 0. Cela signifie : si la boucle termine, alors on a bien x ≥ 0 ∧ x ≥ b. La correction totale (garantie de terminaison) est traitée en Q2 avec le variant b − x.

Invariance 1 pt formule (I∧E) ⊃ [S]I correctement posée · 2 pts calcul [x:=x+1]I par substitution · 2 pts vérification de l'implication 3 c) 4. Q2 10 pts Avancé

On reprend la même boucle. On a établi en Q1 sa correction partielle. L'objectif est maintenant d'établir sa correction totale, c'est-à-dire de prouver qu'elle se termine nécessairement.

On choisit le variant : V = b − x.

a) 2. Typage — Montrez que (I ∧ E) ⊃ V ∈ ℕ, c'est-à-dire que b − x est bien un entier naturel (positif) dès qu'on entre dans la boucle. (3 pts)
b) 3. Terminaison — Prouvez que (I ∧ E ∧ V = δ) ⊃ [x := x+1](V < δ), c'est-à-dire :
(x ≥ 0 ∧ x < b ∧ b−x = δ) ⊃ [x := x+1](b−x < δ)
(5 pts)
c) Conclusion — En combinant Q1 et Q2, que peut-on affirmer sur la boucle ? Rédigez la conclusion en langage naturel. (2 pts)
Piège courant — le rôle de δ : δ est une variable fantôme (constante arbitraire) qui représente la valeur du variant juste avant une itération. Elle ne varie pas pendant l'itération : elle sert de « témoin » pour comparer avant et après. Ne confondez pas δ avec V (qui change) : V = δ est vrai à l'entrée du corps, et on doit montrer V < δ à la sortie.
Pour montrer (I ∧ E) ⊃ V ∈ ℕ, instanciez : I = (x ≥ 0), E = (x < b), V = b − x. Demandez-vous : si x ≥ 0 et x < b, alors que vaut b − x ? Est-il positif ? Est-il entier (si x et b sont des entiers) ?
Pour prouver (x ≥ 0 ∧ x < b ∧ b−x = δ) ⊃ [x := x+1](b−x < δ), calculez [x := x+1](x ≥ 0 ∧ b−x < δ) en substituant x par (x+1) dans la postcondition. Vous obtiendrez deux conditions ; vérifiez que chacune est impliquée par la précondition.
Après substitution, la postcondition devient x+1 ≥ 0 ∧ b−(x+1) < δ, soit x ≥ −1 ∧ b−x−1 < δ.
  • Pour la première partie : la précondition donne x ≥ 0, or 0 > −1, donc x ≥ 0 ⊃ x ≥ −1. ✓
  • Pour la seconde : la précondition donne b−x = δ, donc b−x−1 = δ−1 < δ. ✓

Solution — Q2 (Correction totale)

Données : I = (x ≥ 0) · E = (x < b) · V = b − x · Corps S = (x := x+1).

a) 2. Typage du variant — 3 pts

Formule à prouver : (I ∧ E) ⊃ V ∈ ℕ

Instance : (x ≥ 0 ∧ x < b) ⊃ (b − x > 0 ∧ b − x ∈ ℕ)

  • Positivité : si x < b alors b − x > 0 — par définition de l'inégalité stricte. ✓
  • Intégrité : b et x sont des entiers, donc b − x est un entier. ✓
  • Les deux conditions réunies donnent b − x ∈ ℕ (entier ≥ 1). ✓

b) 3. Terminaison du variant — 5 pts

Formule à prouver : (I ∧ E ∧ V = δ) ⊃ [x := x+1](V < δ)

(x ≥ 0  ∧  x < b  ∧  b−x = δ)  ⊃  [x := x+1](b−x < δ)

Étape 1 — Calcul de [x := x+1](b−x < δ) par substitution

On substitue x par (x+1) dans b−x < δ :

[x := x+1](b−x < δ)
= b−(x+1) < δ
= b−x−1 < δ

Étape 2 — Vérification que la précondition implique le résultat

On doit montrer : (x ≥ 0 ∧ x < b ∧ b−x = δ) ⊃ (b−x−1 < δ)

  • La précondition donne b−x = δ, donc b−x−1 = δ−1 < δ (car 1 > 0). ✓

L'implication est vérifiée, donc la terminaison est prouvée. ✓

c) Conclusion — 2 pts

On a prouvé les deux propriétés nécessaires à la correction totale :

PropriétéProuvée enRésultat
Correction partielle Q1 Si ça termine → la postcondition est vraie ✓
Terminaison Q2 a) + b) V = b−x ∈ ℕ et décroît strictement → ça termine ✓

Conclusion : la boucle while x < b do x := x+1 est totalement correcte : elle se termine toujours (en au plus b − x₀ itérations, où x₀ est la valeur initiale de x), et à la sortie on a bien x ≥ 0 ∧ x ≥ b.
