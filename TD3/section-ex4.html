Exercice 4 — Obligations de preuve d'une boucle

Avancé ◎ 10 pts ⏱ 15–20 min
Objectifs
  • Comprendre le rôle d'un invariant de boucle dans une preuve de correction
  • Formuler les trois obligations de preuve associées à un invariant
  • Vérifier chaque obligation par calcul de wp ou implication logique
  • Distinguer correction partielle et correction totale
  • Construire un variant et prouver la terminaison d'une boucle

Prérequis : Cours 2 — Logique de Hoare · Exercice 3 (wp et validité de triplets) · Notion de variant (terminaison)

Programme à prouver

On considère la boucle suivante, où b est une constante entière quelconque :

{ x >= 0 }
while x < b do
    x := x + 1
done
{ x >= 0  ∧  ¬(x < b) }

L'invariant proposé est I : x ≥ 0. Cet exercice se déroule en deux parties :

  • Q1 — Correction partielle : formuler et vérifier les trois obligations de preuve qui établissent que si la boucle termine, alors la postcondition est satisfaite.
  • Q2 — Correction totale : trouver un variant et prouver que la boucle se termine nécessairement.

Pour prouver {P} while B do S done {Q} en logique de Hoare, on choisit un prédicat I (l'invariant) et on établit trois obligations :

Obligation 1 — Initialisation

P ⟹ I

La précondition doit impliquer l'invariant : I est vrai dès l'entrée dans la boucle.

Obligation 2 — Conservation

{ I ∧ B } S { I }

Si l'invariant est vrai et que la condition est vraie, alors après une itération l'invariant est à nouveau vrai. On prouve ce triplet via wp :

(I ∧ B) ⟹ wp(S, I)

Obligation 3 — Postcondition

(I ∧ ¬B) ⟹ Q

À la sortie de la boucle, la condition est fausse et l'invariant est encore vrai. Ces deux faits réunis doivent impliquer la postcondition attendue.

ObligationFormule généraleQuestion posée
Initialisation P ⟹ I L'invariant est-il vrai avant d'entrer ?
Conservation (I ∧ B) ⟹ wp(S, I) Une itération préserve-t-elle I ?
Postcondition (I ∧ ¬B) ⟹ Q La sortie garantit-elle Q ?

Correction partielle vs totale

Ces trois obligations établissent la correction partielle : si la boucle termine, alors la postcondition est satisfaite. La correction totale (garantie de terminaison) nécessite en plus un variant — voir Q2 ci-dessous.

Correction totale — Variant

La correction totale = correction partielle + terminaison. Pour prouver la terminaison, on introduit un variant V : une expression entière qui décroît strictement à chaque itération et reste positive ou nulle (i.e., V ∈ ℕ).

La notation Δ (delta) représente la valeur du variant avant une itération — c'est une constante figée le temps d'analyser un tour de boucle.

Obligation de typage

I ∧ C ⟹ V ∈ ℕ

Quand on entre dans la boucle (I vrai et C vrai), le variant est un entier naturel (positif ou nul). Cela garantit qu'il ne peut pas décroître indéfiniment.

Obligation de décroissance

{ I ∧ C ∧ V = Δ } S { I ∧ V < Δ }

Si l'invariant est vrai, la condition est vraie, et le variant vaut Δ avant l'itération, alors après l'exécution du corps S, le variant est strictement inférieur à Δ. On prouve ce triplet comme d'habitude par calcul de wp.

ObligationFormuleCe qu'elle garantit
Typage I ∧ C ⟹ V ∈ ℕ Le variant est borné par zéro (ne peut pas être négatif)
Décroissance { I ∧ C ∧ V = Δ } S { I ∧ V < Δ } Le variant diminue strictement à chaque tour
Boucle totale { I } while C do S { I ∧ ¬C } Synthèse : correction partielle + terminaison = correction totale

Analogie

Le variant comme compte à rebours : imaginez un minuteur réglé sur V secondes. À chaque itération, le minuteur doit avancer — il ne peut pas rester immobile. Comme il ne peut pas descendre en dessous de 0, la sonnerie finira forcément par retentir. La boucle est garantie de s'arrêter.

Exemple guidé

Boucle : { n ≥ 0 } while n > 0 do n := n − 1 done { n = 0 }, invariant I : n ≥ 0.

  • Init : n ≥ 0 ⟹ n ≥ 0 — trivial ✓
  • Cons. : wp(n := n−1, n ≥ 0) = n−1 ≥ 0 = n ≥ 1. Or (n ≥ 0 ∧ n > 0) = n ≥ 1 ✓
  • Post : (n ≥ 0 ∧ ¬(n > 0)) = n = 0 ⟹ n = 0 ✓

Analogie

L'invariant comme règle d'entreprise : imaginez une chaîne de montage qui doit respecter la règle "aucune pièce défectueuse ne circule". L'initialisation vérifie cette règle à l'ouverture. La conservation vérifie qu'aucune opération ne l'enfreint. La postcondition vérifie que la règle suffit à garantir la qualité du produit fini.

Q1 10 pts Avancé

Pour la boucle présentée en contexte, avec l'invariant I : x ≥ 0, formulez et vérifiez les trois obligations de preuve.

a) Obligation d'initialisation — montrez que P ⟹ I. (2 pts)
b) Obligation de conservation — montrez que { I ∧ B } x := x+1 { I } est valide. (5 pts)
c) Obligation de postcondition — montrez que (I ∧ ¬B) ⟹ Q. (3 pts)
Piège courant : Confondre l'obligation de conservation et celle de postcondition. La conservation utilise I ∧ B (condition vraie, on est encore dans la boucle). La postcondition utilise I ∧ ¬B (condition fausse, on vient d'en sortir).
Instanciez les trois formules générales avec les valeurs du problème : P = (x ≥ 0), B = (x < b), S = (x := x+1), I = (x ≥ 0), Q = (x ≥ 0 ∧ ¬(x < b)). Les obligations a) et c) sont des implications simples à vérifier directement.
Pour l'obligation b), calculez wp(x := x+1, x ≥ 0) par substitution : remplacez x par (x+1) dans x ≥ 0. Vous obtenez x+1 ≥ 0, soit x ≥ −1. Vérifiez ensuite que (x ≥ 0 ∧ x < b) ⟹ x ≥ −1.

Solution — Q1

Données : P = (x ≥ 0), B = (x < b), S = (x := x+1), I = (x ≥ 0), Q = (x ≥ 0 ∧ ¬(x < b)).

a) Initialisation — 2 pts

Formule : P ⟹ I

Instance : x ≥ 0 ⟹ x ≥ 0

Implication triviale par réflexivité. ✓

b) Conservation — 5 pts

Formule : { I ∧ B } S { I }, soit { x ≥ 0 ∧ x < b } x := x+1 { x ≥ 0 }

On réduit via wp :

  • Calculer wp(x := x+1, x ≥ 0)
    = substituer x par (x+1) dans x ≥ 0
    = x+1 ≥ 0
    = x ≥ −1
  • Vérifier (x ≥ 0 ∧ x < b) ⟹ x ≥ −1
    La conjonction implique x ≥ 0, et x ≥ 0 ⟹ x ≥ −1 car 0 > −1. ✓

c) Postcondition — 3 pts

Formule : (I ∧ ¬B) ⟹ Q

Instance : (x ≥ 0 ∧ ¬(x < b)) ⟹ (x ≥ 0 ∧ ¬(x < b))

Implication triviale : la postcondition Q est exactement I ∧ ¬B. ✓

Conclusion — Correction partielle établie

Les trois obligations sont vérifiées : la boucle est partiellement correcte vis-à-vis de l'invariant x ≥ 0. Cela signifie : si la boucle termine, alors on a bien x ≥ 0 ∧ x ≥ b. La correction totale (garantie de terminaison) est traitée en Q2 avec le variant b − x.

2 a) Initialisation 1 pt formule P ⟹ I correctement instanciée · 1 pt justification de l'implication 5 b) Conservation 1 pt triplet {I∧B} S {I} correctement posé · 2 pts calcul wp · 2 pts vérification de l'implication 3 c) Postcondition 1 pt formule (I∧¬B) ⟹ Q instanciée · 2 pts identification Q = I∧¬B et conclusion
Sous-total Q1 : 10 pts
Q2 10 pts Avancé

On reprend la même boucle. On a établi en Q1 sa correction partielle. L'objectif est maintenant d'établir sa correction totale, c'est-à-dire de prouver qu'elle se termine nécessairement.

On choisit le variant : V = b − x.

a) Typage — Montrez que I ∧ C ⟹ V ∈ ℕ, c'est-à-dire que b − x est bien un entier naturel (positif) dès qu'on entre dans la boucle. (3 pts)
b) Décroissance — Prouvez le triplet { I ∧ C ∧ V = Δ } x := x+1 { I ∧ V < Δ }, c'est-à-dire :
{ x ≥ 0 ∧ x < b ∧ b−x = Δ }   x := x+1   { x ≥ 0 ∧ b−x < Δ }
(5 pts)
c) Conclusion — En combinant Q1 et Q2, que peut-on affirmer sur la boucle ? Rédigez la conclusion en langage naturel. (2 pts)
Piège courant — le rôle de Δ : Δ est une constante arbitraire qui représente la valeur du variant juste avant une itération. Elle ne varie pas pendant l'itération : elle sert de « témoin » pour comparer avant et après. Ne confondez pas Δ avec V (qui change) : V = Δ est vrai à l'entrée du corps, et on doit montrer V < Δ à la sortie.
Pour montrer I ∧ C ⟹ V ∈ ℕ, instanciez : I = (x ≥ 0), C = (x < b), V = b − x. Demandez-vous : si x ≥ 0 et x < b, alors que vaut b − x ? Est-il positif ? Est-il entier (si x et b sont des entiers) ?
Pour prouver { x ≥ 0 ∧ x < b ∧ b−x = Δ } x := x+1 { x ≥ 0 ∧ b−x < Δ }, utilisez la règle d'affectation : calculez wp(x := x+1,   x ≥ 0 ∧ b−x < Δ) en substituant x par (x+1) dans la postcondition. Vous obtiendrez deux conditions ; vérifiez que chacune est impliquée par la précondition.
Après substitution, la postcondition devient x+1 ≥ 0 ∧ b−(x+1) < Δ, soit x ≥ −1 ∧ b−x−1 < Δ.
  • Pour la première partie : la précondition donne x ≥ 0, or 0 > −1, donc x ≥ 0 ⟹ x ≥ −1. ✓
  • Pour la seconde : la précondition donne b−x = Δ, donc b−x−1 = Δ−1 < Δ. ✓

Solution — Q2 (Correction totale)

Données : I = (x ≥ 0) · C = (x < b) · V = b − x · Corps S = (x := x+1).

a) Typage du variant — 3 pts

Formule à prouver : I ∧ C ⟹ V ∈ ℕ

Instance : x ≥ 0 ∧ x < b  ⟹  b − x > 0 ∧ b − x ∈ ℕ

  • Positivité : si x < b alors b − x > 0 — par définition de l'inégalité stricte. ✓
  • Intégrité : b et x sont des entiers, donc b − x est un entier. ✓
  • Les deux conditions réunies donnent b − x ∈ ℕ (entier ≥ 1). ✓

b) Décroissance du variant — 5 pts

Triplet à prouver :

{ x ≥ 0  ∧  x < b  ∧  b−x = Δ }   x := x+1   { x ≥ 0  ∧  b−x < Δ }

Étape 1 — Calcul de wp par la règle d'affectation

On substitue x par (x+1) dans la postcondition x ≥ 0 ∧ b−x < Δ :

wp(x := x+1,  x ≥ 0 ∧ b−x < Δ)
= [x+1 / x]( x ≥ 0 ∧ b−x < Δ )
= (x+1) ≥ 0 ∧ b−(x+1) < Δ
= x ≥ −1 ∧ b−x−1 < Δ

Étape 2 — Vérification que la précondition implique la wp

On doit montrer : (x ≥ 0 ∧ x < b ∧ b−x = Δ) ⟹ (x ≥ −1 ∧ b−x−1 < Δ)

  • Partie gauche x ≥ −1 : la précondition donne x ≥ 0. Or 0 > −1, donc x ≥ 0 ⟹ x ≥ −1. ✓
  • Partie droite b−x−1 < Δ : la précondition donne b−x = Δ, donc b−x−1 = Δ−1 < Δ (car 1 > 0). ✓

Les deux parties sont vérifiées, donc le triplet est valide. ✓

c) Conclusion — 2 pts

On a prouvé les deux propriétés nécessaires à la correction totale :

PropriétéProuvée enRésultat
Correction partielle Q1 Si ça termine → la postcondition est vraie ✓
Terminaison Q2 a) + b) V = b−x ∈ ℕ et décroît strictement → ça termine ✓

Conclusion : la boucle while x < b do x := x+1 est totalement correcte : elle se termine toujours (en au plus b − x₀ itérations, où x₀ est la valeur initiale de x), et à la sortie on a bien x ≥ 0 ∧ x ≥ b.
