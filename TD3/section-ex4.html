Exercice 4 — Obligations de preuve d'une boucle

Avancé ◎ 10 pts ⏱ 15–20 min
Objectifs
  • Comprendre le rôle d'un invariant de boucle dans une preuve de correction
  • Formuler les trois obligations de preuve associées à un invariant
  • Vérifier chaque obligation par calcul de wp ou implication logique

Prérequis : Cours 2 — Logique de Hoare · Exercice 3 (wp et validité de triplets)

Programme à prouver

On considère la boucle suivante, où b est une constante entière quelconque :

{ x >= 0 }
while x < b do
    x := x + 1
done
{ x >= 0  ∧  ¬(x < b) }

L'invariant proposé est I : x ≥ 0. Votre objectif est de formuler et de vérifier les trois obligations de preuve qui établissent la correction partielle de cette boucle vis-à-vis de cet invariant.

Pour prouver {P} while B do S done {Q} en logique de Hoare, on choisit un prédicat I (l'invariant) et on établit trois obligations :

Obligation 1 — Initialisation

P ⟹ I

La précondition doit impliquer l'invariant : I est vrai dès l'entrée dans la boucle.

Obligation 2 — Conservation

{ I ∧ B } S { I }

Si l'invariant est vrai et que la condition est vraie, alors après une itération l'invariant est à nouveau vrai. On prouve ce triplet via wp :

(I ∧ B) ⟹ wp(S, I)

Obligation 3 — Postcondition

(I ∧ ¬B) ⟹ Q

À la sortie de la boucle, la condition est fausse et l'invariant est encore vrai. Ces deux faits réunis doivent impliquer la postcondition attendue.

ObligationFormule généraleQuestion posée
Initialisation P ⟹ I L'invariant est-il vrai avant d'entrer ?
Conservation (I ∧ B) ⟹ wp(S, I) Une itération préserve-t-elle I ?
Postcondition (I ∧ ¬B) ⟹ Q La sortie garantit-elle Q ?

Correction partielle vs totale

Ces trois obligations établissent la correction partielle : si la boucle termine, alors la postcondition est satisfaite. La correction totale (garantie de terminaison) nécessite en plus un variant — un entier strictement décroissant à chaque itération.

Exemple guidé

Boucle : { n ≥ 0 } while n > 0 do n := n − 1 done { n = 0 }, invariant I : n ≥ 0.

  • Init : n ≥ 0 ⟹ n ≥ 0 — trivial ✓
  • Cons. : wp(n := n−1, n ≥ 0) = n−1 ≥ 0 = n ≥ 1. Or (n ≥ 0 ∧ n > 0) = n ≥ 1 ✓
  • Post : (n ≥ 0 ∧ ¬(n > 0)) = n = 0 ⟹ n = 0 ✓

Analogie

L'invariant comme règle d'entreprise : imaginez une chaîne de montage qui doit respecter la règle "aucune pièce défectueuse ne circule". L'initialisation vérifie cette règle à l'ouverture. La conservation vérifie qu'aucune opération ne l'enfreint. La postcondition vérifie que la règle suffit à garantir la qualité du produit fini.

Q1 10 pts Avancé

Pour la boucle présentée en contexte, avec l'invariant I : x ≥ 0, formulez et vérifiez les trois obligations de preuve.

a) Obligation d'initialisation — montrez que P ⟹ I. (2 pts)
b) Obligation de conservation — montrez que { I ∧ B } x := x+1 { I } est valide. (5 pts)
c) Obligation de postcondition — montrez que (I ∧ ¬B) ⟹ Q. (3 pts)
Piège courant : Confondre l'obligation de conservation et celle de postcondition. La conservation utilise I ∧ B (condition vraie, on est encore dans la boucle). La postcondition utilise I ∧ ¬B (condition fausse, on vient d'en sortir).
Instanciez les trois formules générales avec les valeurs du problème : P = (x ≥ 0), B = (x < b), S = (x := x+1), I = (x ≥ 0), Q = (x ≥ 0 ∧ ¬(x < b)). Les obligations a) et c) sont des implications simples à vérifier directement.
Pour l'obligation b), calculez wp(x := x+1, x ≥ 0) par substitution : remplacez x par (x+1) dans x ≥ 0. Vous obtenez x+1 ≥ 0, soit x ≥ −1. Vérifiez ensuite que (x ≥ 0 ∧ x < b) ⟹ x ≥ −1.

Solution — Q1

Données : P = (x ≥ 0), B = (x < b), S = (x := x+1), I = (x ≥ 0), Q = (x ≥ 0 ∧ ¬(x < b)).

a) Initialisation — 2 pts

Formule : P ⟹ I

Instance : x ≥ 0 ⟹ x ≥ 0

Implication triviale par réflexivité. ✓

b) Conservation — 5 pts

Formule : { I ∧ B } S { I }, soit { x ≥ 0 ∧ x < b } x := x+1 { x ≥ 0 }

On réduit via wp :

  • Calculer wp(x := x+1, x ≥ 0)
    = substituer x par (x+1) dans x ≥ 0
    = x+1 ≥ 0
    = x ≥ −1
  • Vérifier (x ≥ 0 ∧ x < b) ⟹ x ≥ −1
    La conjonction implique x ≥ 0, et x ≥ 0 ⟹ x ≥ −1 car 0 > −1. ✓

c) Postcondition — 3 pts

Formule : (I ∧ ¬B) ⟹ Q

Instance : (x ≥ 0 ∧ ¬(x < b)) ⟹ (x ≥ 0 ∧ ¬(x < b))

Implication triviale : la postcondition Q est exactement I ∧ ¬B. ✓

Conclusion

Les trois obligations sont vérifiées : la boucle est partiellement correcte vis-à-vis de l'invariant x ≥ 0. La terminaison (correction totale) n'est pas demandée ici, mais s'établirait avec le variant b − x, strictement décroissant à chaque itération tant que x < b.
