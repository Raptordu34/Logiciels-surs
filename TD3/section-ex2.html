Exercice 2 — Règles d'exécution

Intermédiaire ◎ 15 pts ⏱ 15–20 min
Objectifs
  • Distinguer évaluation d'expression (→ valeur) et exécution d'instruction (→ nouvel état)
  • Appliquer les règles d'affectation, de conditionnelle et de boucle
  • Construire des arbres de dérivation pour des programmes complets

Prérequis : Cours 1 — Sémantique opérationnelle · Exercice 1 (évaluation d'expressions)

Du calcul de valeur à la transformation d'état

Dans l'exercice 1, les expressions produisaient une valeur : ⟨E, s⟩ → v. Les instructions produisent quant à elles un nouvel état : ⟨I, s⟩ → s'. L'affectation, le conditionnel et la boucle modifient la mémoire — ils ne calculent pas.

Chaque instruction possède une ou plusieurs règles de dérivation. Les prémisses sont au-dessus de la barre, la conclusion en dessous.

Affectation

Si ⟨E, s⟩ → v, alors ⟨x := E, s⟩ → s[x ↦ v]

Une seule prémisse : évaluer l'expression E, puis mettre à jour l'état.

Conditionnelle — deux règles

  • IF-Vrai : si ⟨B, s⟩ → Vrai et ⟨I₁, s⟩ → s',
    alors ⟨if B then I₁ else I₂, s⟩ → s'
  • IF-Faux : si ⟨B, s⟩ → Faux et ⟨I₂, s⟩ → s',
    alors ⟨if B then I₁ else I₂, s⟩ → s'

On trace une seule branche selon la valeur de B — jamais les deux.

Boucle — deux règles

  • WHILE-Faux : si ⟨B, s⟩ → Faux,
    alors ⟨while B do I, s⟩ → s (l'état ne change pas)
  • WHILE-Vrai : si ⟨B, s⟩ → Vrai, ⟨I, s⟩ → s' et ⟨while B do I, s'⟩ → s'',
    alors ⟨while B do I, s⟩ → s''

WHILE-Vrai a trois prémisses. La troisième est récursive : on relance le même while dans le nouvel état.

InstructionPrémissesConclusion
x := E ⟨E, s⟩ → v ⟨x := E, s⟩ → s[x ↦ v]
if B then I₁ else I₂ ⟨B, s⟩ → Vrai/Faux + branche correspondante ⟨if…, s⟩ → s'
while B do I (arrêt) ⟨B, s⟩ → Faux ⟨while…, s⟩ → s
while B do I (itération) ⟨B,s⟩→Vrai + ⟨I,s⟩→s' + ⟨while…,s'⟩→s'' ⟨while…, s⟩ → s''

Analogie

Un état est un instantané de la mémoire. Chaque instruction produit un nouvel instantané. L'arbre de dérivation est le journal de toutes les transformations successives — comme un historique de commits git, chaque état dérivant du précédent.

Exemple guidé — Affectation

Exécuter y := y + 2 dans l'état s avec s(y) = 5.

  • Prémisse : ⟨y + 2, s⟩ → 7 (règle op. binaire sur ⟨y,s⟩→5 et ⟨2,s⟩→2)
  • Conclusion — règle Affectation : ⟨y := y + 2, s⟩ → s[y ↦ 7]

L'état résultant n'est pas "7" — c'est un état complet où y vaut 7.

Méthode générale

  • Identifier la règle principale (affectation / if / while).
  • Lister ses prémisses dans l'ordre.
  • Construire chaque prémisse (souvent un sous-arbre d'expression).
  • Écrire la conclusion avec le bon état résultant.
Q1 5 pts Fondamental

Appliquez les règles de la sémantique opérationnelle pour exécuter :
x := x + 1 dans l'état s tel que s(x) = 2.

Piège courant : Écrire l'état résultant comme une valeur seule (3). L'affectation modifie l'état — la conclusion doit être ⟨x := x+1, s⟩ → s[x ↦ 3], pas "x vaut 3".
La règle Affectation a une seule prémisse : évaluer l'expression à droite de :=. Construisez d'abord le sous-arbre pour ⟨x + 1, s⟩ → ?.
⟨x + 1, s⟩ → 3 car s(x)=2 et 2+1=3. Appliquez maintenant la règle Affectation : ⟨x := x+1, s⟩ → s[x ↦ 3].

Solution — Q1

⟨x, s⟩ → 2 ⟨1, s⟩ → 1
⟨x + 1, s⟩ → 3
⟨x := x + 1, s⟩ → s[x ↦ 3]
Règles appliquées
  • ⟨x, s⟩ → 2 — règle Variable : s(x) = 2
  • ⟨1, s⟩ → 1 — règle Entier
  • ⟨x+1, s⟩ → 3 — règle Op. binaire (+) : 2+1 = 3
  • ⟨x := x+1, s⟩ → s[x ↦ 3] — règle Affectation
Q2 5 pts Intermédiaire

Appliquez les règles de la sémantique opérationnelle pour exécuter :
if x < 0 then x := x − 1 else x := x + 1 dans l'état s tel que s(x) = 5.

Piège courant : Tracer les deux branches (then et else) dans l'arbre. La règle conditionnelle sélectionne une seule branche selon la valeur de vérité de la condition. L'autre branche n'apparaît pas dans la dérivation.
Évaluez d'abord la condition x < 0 dans l'état s avec s(x) = 5. Est-ce Vrai ou Faux ? Quelle règle (IF-Vrai ou IF-Faux) s'applique ?
5 < 0 est Faux → règle IF-Faux. Les deux prémisses sont : (1) ⟨x < 0, s⟩ → Faux et (2) ⟨x := x + 1, s⟩ → ?. Construisez chacune de ces deux prémisses, puis écrivez la conclusion.

Solution — Q2

La condition x < 0 est Fausse (5 < 0 = Faux) → règle IF-Faux, deux prémisses :

⟨x, s⟩ → 5 ⟨0, s⟩ → 0
⟨x < 0, s⟩ → Faux
⟨x, s⟩ → 5 ⟨1, s⟩ → 1
⟨x + 1, s⟩ → 6
⟨x := x + 1, s⟩ → s[x ↦ 6]
⟨if x < 0 then x := x−1 else x := x+1, s⟩ → s[x ↦ 6]
Règles appliquées
  • Prémisse gauche — règle Op. binaire (<) sur ⟨x,s⟩→5 et ⟨0,s⟩→0 → Faux
  • Prémisse droite — règle Affectation sur la branche else (x := x+1 → s[x↦6])
  • Conclusion — règle IF-Faux : état résultant s[x ↦ 6]

Méthode — Boucle while

  • Nommer les états intermédiaires : s₀ (état initial), s₁ (après le corps), etc.
  • Évaluer la condition dans l'état courant — Vrai → règle WHILE-Vrai (3 prémisses), Faux → règle WHILE-Faux (1 prémisse).
  • Construire le sous-arbre du corps (affectation ou séquence).
  • Ajouter la troisième prémisse : relancer récursivement le while dans le nouvel état.
Q3 5 pts Avancé

Appliquez les règles de la sémantique opérationnelle pour exécuter :
while x < 100 do x := x * 2 dans l'état s tel que s(x) = 50.

Piège courant : Oublier la troisième prémisse de WHILE-Vrai. La règle exige de prouver que le while termine aussi dans le nouvel état s₁. Sans cette prémisse récursive, la preuve est incomplète.
Notez s₀ = s (avec s₀(x) = 50) et s₁ = s[x ↦ 100] (état après le corps). La boucle s'exécute une seule fois : vérifiez pourquoi la condition est vraie en s₀ et fausse en s₁.
L'arbre principal applique WHILE-Vrai avec trois prémisses côte à côte : (1) ⟨x < 100, s₀⟩ → Vrai, (2) ⟨x := x*2, s₀⟩ → s₁, (3) ⟨while x < 100 do…, s₁⟩ → s₁ — cette dernière utilise WHILE-Faux car 100 < 100 est Faux.

Solution — Q3

On pose s₀ = s (s₀(x) = 50) et s₁ = s[x ↦ 100].

La condition est vraie en s₀ (50 < 100) → règle WHILE-Vrai, trois prémisses.

⟨x, s₀⟩ → 50 ⟨100, s₀⟩ → 100
⟨x < 100, s₀⟩ → Vrai
⟨x, s₀⟩ → 50 ⟨2, s₀⟩ → 2
⟨x * 2, s₀⟩ → 100
⟨x := x * 2, s₀⟩ → s₁
⟨x, s₁⟩ → 100 ⟨100, s₁⟩ → 100
⟨x < 100, s₁⟩ → Faux
⟨while x < 100 do…, s₁⟩ → s₁
⟨while x < 100 do x := x * 2, s₀⟩ → s₁
Règles appliquées
  • Prémisse 1 — règle Op. binaire (<) : 50 < 100 → Vrai
  • Prémisse 2 — règles Op. binaire (*) puis Affectation : s₀ → s₁ = s[x ↦ 100]
  • Prémisse 3 — règle WHILE-Faux dans s₁ : 100 < 100 → Faux, état inchangé
  • Conclusion — règle WHILE-Vrai : état final s₁ = s[x ↦ 100]
