Exercice 3 — Preuve de validité

Intermédiaire ◎ 10 pts ⏱ 15–20 min
Objectifs
  • Calculer la précondition la plus faible (wp) d'une affectation
  • Appliquer la règle de séquence pour une suite d'instructions
  • Conclure à la validité d'un triplet de Hoare par implication logique

Prérequis : Cours 2 — Logique de Hoare · Axiome de l'affectation · Règle de séquence

Un triplet de Hoare {P} C {Q} est valide si : chaque fois que P est vraie avant l'exécution de C, alors Q est vraie après.

Précondition la plus faible — wp

wp(C, Q) est la condition minimale à garantir avant C pour être certain que Q soit vraie après. Méthode de preuve :

  • Calculer wp(C, Q).
  • Vérifier que P ⟹ wp(C, Q).
  • Si l'implication est vraie, le triplet {P} C {Q} est valide.

Axiome de l'affectation

wp(x := E, Q) = Q[E/x]

On remplace toutes les occurrences de x par l'expression E dans la postcondition Q. La substitution s'effectue dans la postcondition, jamais dans la précondition.

Règle de séquence

Pour {P} C₁ ; C₂ {Q}, on travaille de droite à gauche :

  • Calculer d'abord R = wp(C₂, Q).
  • Puis calculer wp(C₁, R).
  • Vérifier que P ⟹ wp(C₁, R).

On remonte le programme en partant du résultat attendu — comme lire une recette à l'envers pour savoir ce qu'il faut préparer.

RègleFormuleCondition de validité
Affectation wp(x := E, Q) = Q[E/x] P ⟹ Q[E/x]
Séquence wp(C₁;C₂, Q) = wp(C₁, wp(C₂, Q)) P ⟹ wp(C₁, wp(C₂, Q))
Conséquence {P'} C {Q'} valide P ⟹ P' et Q' ⟹ Q

Exemple guidé

Prouver { y > 7 } y := y + 3 { y > 10 }.

  • Calculer wp : wp(y := y+3, y > 10) = (y+3) > 10 = y > 7
  • Vérifier : précondition donnée = y > 7 ✓
  • L'implication y > 7 ⟹ y > 7 est vraie → triplet valide.

Analogie

wp comme exigence minimale d'entrée : imaginez une machine qui garantit "la pièce sortira conforme" (postcondition Q). wp(C, Q) est la liste minimale des contrôles à faire sur la matière première (précondition) pour que la garantie tienne. Si votre fournisseur vous livre mieux que ce minimum, c'est encore valide.

Q1 5 pts Fondamental

Prouvez la validité du triplet de Hoare :
{ x > a }   x := x + 1   { x > a + 1 }

Piège courant : Substituer dans la précondition plutôt que dans la postcondition. L'axiome de l'affectation dit wp(x := E, Q) = Q[E/x] — la substitution s'opère toujours sur Q (postcondition), pas sur P.
Calculez wp(x := x+1, x > a+1) en remplaçant toutes les occurrences de x par (x+1) dans la postcondition x > a+1.
Après substitution : (x+1) > a+1, ce qui se simplifie en x > a. La précondition donnée est exactement x > a. Concluez par l'implication x > a ⟹ x > a.

Solution — Q1

  • Calculer wp : 
    wp(x := x+1,  x > a+1)
= [x+1 / x](x > a+1)
= (x+1) > a+1
= x > a
  • Vérifier l'implication :

    Précondition donnée : x > a

    wp calculée : x > a

    x > a ⟹ x > a — triviale, toujours vraie. ✓

  • Conclusion : le triplet { x > a } x := x+1 { x > a+1 } est valide.
Reformulation sous forme de triplet prouvé

On a montré que x > a ⟹ wp(x := x+1, x > a+1), ce qui, par la règle de conséquence, établit la validité du triplet.

Méthode — Séquence

  • Partir de la postcondition finale Q.
  • Calculer wp de la dernière instruction par rapport à Q → obtenir R.
  • Calculer wp de la première instruction par rapport à R → obtenir wp global.
  • Vérifier que P ⟹ wp global.
Q2 5 pts Intermédiaire

Prouvez la validité du triplet de Hoare :
{ x > a }   x := x + 1 ; x := x + x   { x > 2·a + 1 }

Piège courant : Travailler de gauche à droite (appliquer wp à x := x+1 en premier). La règle de séquence se calcule de droite à gauche : on commence par la dernière instruction et on remonte.
Commencez par la dernière instruction : calculez wp(x := x+x,   x > 2·a+1). Remplacez x par (x+x), c'est-à-dire 2x, dans la postcondition.
wp(x := x+x, x > 2·a+1) = 2x > 2·a+1. Prenez ce résultat comme nouvelle postcondition et calculez wp(x := x+1,   2x > 2·a+1) en remplaçant x par (x+1). Simplifiez l'inégalité obtenue, puis vérifiez que x > a l'implique.

Solution — Q2

  • Étape 1 — wp de la dernière instruction 
    wp(x := x+x,  x > 2·a+1)
= [x+x / x](x > 2·a+1)
= 2x > 2·a+1
  • Étape 2 — wp de la première instruction 
    wp(x := x+1,  2x > 2·a+1)
= [x+1 / x](2x > 2·a+1)
= 2·(x+1) > 2·a+1
= 2x + 2 > 2·a + 1
= 2x > 2·a - 1
= x > a - ½
  • Étape 3 — Vérifier l'implication

    Précondition donnée : x > a

    wp global calculé : x > a − ½

    x > a ⟹ x > a − ½ — vraie car a > a − ½. ✓

  • Conclusion : le triplet est valide.
Récapitulatif des états intermédiaires

Après x := x+xx > 2·a+1 (postcondition)
Pourquoi l'implication fonctionne-t-elle ?

La wp calculée (x > a−½) est une condition plus faible que la précondition donnée (x > a). La règle de conséquence autorise une précondition plus forte que la wp minimale : on garantit plus que le nécessaire, ce qui suffit.
